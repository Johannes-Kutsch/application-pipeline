# Writing Style

Use this file for declarative voice and strategy rules only. Concrete exemplars do not belong here. Put positive-only snippets in `positive-exemplars.md`.

## Voice

- First person, grounded, specific.
- Clear sentences over buzzwords or inflated claims.

## Do

- Prefer concrete claims that can be backed by facts from the profile or analysis.
- Keep paragraphs compact and purposeful.

## Don't

- Avoid hollow adjectives and generic motivation phrasing.
- Avoid sounding like a catalogue of projects or skills.

## Register

- Professional, direct, and human.
- Confident without overselling.

## Cover Strategy

- Use the opener for one listing-specific resonance hook, not a broad self-summary.
- Build the middle around one dominant capability arc with up to two supporting evidence anchors.
- Treat projects as evidence for the arc, not as a list to name-drop.
- Keep each paragraph aligned to a single job-relevant point.
