# Interactive cover pattern library

`/write-cv` will draft cover-letter paragraphs interactively against a global `cover-patterns.md` library instead of relying on one-shot free prose. For each cover slot it may reuse a matching candidate-approved pattern verbatim or lightly adapt it, then asks for confirmation; if no pattern fits or the candidate rejects the match, it proposes alternatives. When the candidate explicitly approves a significantly new slot-purpose plus argument-type combination, `/write-cv` may write it back to `cover-patterns.md`, so the system learns at the point where the quality decision is made.

The interactive flow applies only to the four cover prose slots, not `opening`, resume, or skills. Resume and skills are still assembled automatically before cover drafting, so the cover flow can see which evidence is already present. If compilation shows cover overflow, `/write-cv` uses a separate interactive shortening loop: it proposes full shortened paragraph variants, ideally one per cover slot, the candidate chooses one, and only `cv.tex` changes before rebuild. Post-build shortening does not update `cover-patterns.md`; resume overflow remains automatic. Ad-hoc style signals stay local to the current draft instead of being written to a separate style file.
