# Application Pipeline

Personal job-discovery and triage pipeline. Fetches listings from a small set of sources, classifies relevance with Claude, accumulates an in-domain **Pool** across days, emits one dated **Daily Results File** per day carrying the **Daily Top-5** **Cards** ranked by the **Match Judge**.

## Scope

- **In scope (v1):** Working **Parsers** for **Bundesagentur**, **stellen.hamburg**, **jobs-beim-staat**, smoke-tested standalone on the laptop.
- **In scope (v1.1):** Full pipeline on Pi ‚Äî orchestrator, **LLM Enricher** (ADR-0032 ‚Äî body fetch + strip + **Relevance Classifier** producing **Header** + **Summary**), **Match Judge** (one call per run, picks **Daily Top-5** from the **Pool**), **Deduplication**, daily file, cron once per day, Syncthing sync.
- **Out of scope:** **Profile** ingestion, additional commercial parsers, browser automation. (CV / cover-letter LaTeX compilation in scope via `application-pipeline compile-cv` ‚Äî see Invocation.)

## Language

### Pipeline artifacts

**Config**: `config.py` at `<cwd>/application-pipeline/` (ADR-0022) controlling pipeline shape ‚Äî `SOURCES`, `LOCATIONS`, `INCLUDE_REMOTE`, optional `claude_cli_path`, `MAX_LISTING_AGE_DAYS: int` (default 180, `‚â• 1`) driving the **Freshness Gate**, `claude_classify_parallelism: int` (default 4, `‚â• 1`) sizing the classify worker pool (ADR-0031, ADR-0042), `claude_classify_batch_size: int` (default 10, `‚â• 1`) ‚Äî listings per classify call (ADR-0046), `DEDUP_COOLDOWN_DAYS: int` (default 30, `‚â• 1`) gating decay of `selected_by_judge` and `expired` entries (ADR-0044). No path override knobs (ADR-0008 amended). `layout.py` retired (ADR-0033). `KEYWORDS`/`NEGATIVE_KEYWORDS` live in **SearchTerms** (ADR-0021). Materialised by `init`; never overwritten by `init --refresh`. Loaded by **Config Loader** into frozen typed `Config`. `max_results` retired (ADR-0041). Cron: weekdays 00:30 (ADR-0017). _Avoid_: config file, settings file, search config.

**Init Bootstrap**: Deep implementation behind `application-pipeline init [--refresh]` that materialises package template resources into the local workspace while preserving operator-owned artefacts. Caller-facing interface stays `init(cwd, refresh=False)` / CLI only. Internally it owns seed policy, bucket routing, package-owned versus operator-owned classification, planned seed and cleanup actions, retired refresh paths, legacy `application-pipeline/skills/` cleanup, and normal/refresh console reporting. Package-owned artefacts include setup scripts, CV Skeleton, tool-root-inlined agent skill bodies, tool-root `SKILL.md` files, and package-templated tool-root `_shared` docs. Operator-owned artefacts include Config, SearchTerms, Triage Profile, CV user-info files, `.gitignore`, `.runtime-data/`, user-added Agent Skill dirs, legacy `application-pipeline/agent-skills/`, and unknown files inside package-owned skill or `_shared` dirs. `init` seeds missing package-owned artefacts; `init --refresh` overwrites only package-owned artefacts and does not materialise or delete legacy `application-pipeline/agent-skills/`. Tests cross the Init Bootstrap interface with temporary filesystem state and real package resources; avoid helper-level seams, template storage adapters, and filesystem-write ports.

**SearchTerms**: User-authored search/filter knobs (ADR-0021) ‚Äî `KEYWORDS`, `NEGATIVE_KEYWORDS`. Two markdown files under `<settings-dir>/user-info/search-terms/` (ADR-0024): `keywords.md`, `negative-keywords.md`. Filename *is* the section. Flat `-` bullet entries. `keywords.md` missing/empty ‚Üí `SearchTermsError`; `negative-keywords.md` optional. `KEYWORDS` ‚Üí parser orchestration, `NEGATIVE_KEYWORDS` ‚Üí **Domain Pre-Filter**. `skills.md` relocated to **Triage Profile** dir (#615). _Avoid_: search config, term file.

**Layout**: _Retired_ per ADR-0033. Card structure hardcoded in **DailyResultsFile**. `init --refresh` deletes `layout.py`. _Avoid_: do not reintroduce.

**Daily Results File**: One dated markdown file at `<settings-dir>/results/YYYY-MM-DD.md`, holding **Daily Top-5** as **Cards** in **Rank** order. Date is cron-anchored (ADR-0015/0016). No preamble, no Run Divider. Write-once; synced read-only. _Avoid_: results file (without "daily"), `current.md`.

**Failure Report**: Markdown at `<settings-dir>/.runtime-data/failures/<timestamp>.md` (ADR-0007, path ADR-0037). Triggers: `cron.sh` errors (ADR-0020), orchestrator runtime errors, **Match Judge** failure, parser-dead events (ADR-0041), missing title in discover (ADR-0041), native enrich failures (ADR-0040). Must include enough context to diagnose without `run.log`. The `application_pipeline.failure_report` module owns timestamp generation, package-tag discovery, markdown rendering, temporary-file replacement, and trigger-specific stage/log-tail composition where the trigger crosses its path-bound recording seam. Callers bind the failures directory once per run or invocation where practical; the legacy simple write function may remain as a compatibility wrapper during migration. Acknowledged by deletion. Quota errors do NOT trigger ‚Äî they sleep (ADR-0016). Per-listing soft failures (oversized/malformed) stash to sibling dirs without `seen.json` write. _Avoid_: incident report, error log, failure-report adapter.

**Position**: A single job listing surviving **Relevance Classifier** and **Match Judge** top-5 selection. Card shape fixed (ADR-0033): `# **{rank}:** {Header}` + `{Summary}`. _Avoid_: job, listing, vacancy.

**Position Schema**: _Retired_ per ADR-0032. Closed-enum fields live inside the LLM-authored **Header**. `PositionStub` and `EnrichResult` survive (ADR-0038). "Parsers never guess" invariant retires ‚Äî LLM Enricher infers from body. _Avoid_: do not reintroduce per-field extraction.

**Raw Description**: Full body text, fetched and stripped by parser's `enrich()` (per-source CSS selector or generic library fallback). Fed to LLM and persisted in **Card Store** at classify time. Rendered into **Daily Results File** after **Summary**, fenced by `---` above and below. Empty body dropped by **Content Gate** (ADR-0030). Oversized stashed to `.runtime-data/failures/oversized/`, no `seen.json` mark. _Avoid_: description (when full text is meant).

**Structured Extract**: _Retired_ per ADR-0032. Replaced by **Header** + **Summary** + **Raw Description**. `extracts.json` is `{listing_id: {header, summary, body}}`, keyed by **Listing ID** integer. _Avoid_: do not reintroduce.

**Header**: Three-line block authored by the **LLM Enricher** at classify time (ADR-0032) ‚Äî title, `company ¬∑ location ¬∑ work_model`, `posted_date ¬∑ seniority ¬∑ salary`. LLM substitutes known values, infers from body, or drops segments. Persisted in `extracts.json`. _Avoid_: card top, headline.

**Summary**: Prose paragraph authored at classify time (ADR-0032), describing the role in the **Triage Profile**'s frame. Persisted alongside **Header**. **Match Judge** ranks on Header + Summary directly. _Avoid_: match verdict summary (retired), description (overloaded).

### Filtering & scoring

**Triage Profile**: Pipeline-facing applicant data. Lives in `<settings-dir>/user-info/triage-profile/` (ADR-0024, ADR-0034) as three files: `gate-criteria.md` (flat domain-in/out + hard exclusions ‚Äî classifier only), `candidate-profile.md` (who-the-candidate-is + ranking preferences ‚Äî judge only), `skills.md` (relocated from `search-terms/`, judge `{SKILLS}` slot, #615). `application_pipeline.triage_profile` owns the local filesystem artefact contract for prompt-slot loading: retired filename checks, required `gate-criteria.md` / `candidate-profile.md` reads, optional `skills.md`, and the **TriageSkills** judge projection for `{SKILLS}`. It exposes the prompt-slot values `{GATE_CRITERIA}`, `{CANDIDATE_PROFILE}`, and `{SKILLS}`. The **Prompt Loader** consumes those values and owns prompt template loading, slot validation, brace escaping, and final `PromptTemplate` construction. Classifier consumes `{GATE_CRITERIA}` only; judge consumes `{CANDIDATE_PROFILE}` + `{SKILLS}`. **TriageSkills** supplies the attribute-stripped judge projection for `{SKILLS}` while preserving the grouped projection for CV authoring. Bullets/keywords, German, concise (ADR-0019). Cover-writing style and paragraph-pattern decisions are handled in `/write-cv`, not as separate triage-profile files. _Avoid_: profile (unqualified), bio, CV Profile (retired), `domain-fit.md` (retired), `self-description.md` (retired), `match-criteria.md` (retired).

**TriageSkills**: Domain module owning the `skills.md` **Triage Profile** artefact. Parses local `skills.md` text/files once and exposes two projections: judge-facing flat **Skill** bullet text with trailing `{...}` attributes stripped, and grouped **Skill Group** data with group/item attributes preserved for CV authoring. Missing `skills.md` yields an empty judge-facing view; flat legacy files remain valid for the judge and degenerate to no **Skill Groups** for CV authoring. _Avoid_: prompt-private skills parser, separate skills parser.

### CV authoring

**CV Slot-Map**: Per-listing `<app_dir>/cv.tex` by `/write-cv`, consumed by `compile-cv` (ADR-0023). `^%% SLOT: <name>$` markers, raw TeX bodies. Thirteen slots: `recipient_company/name/street/zip_city`, `opening`, `cover_intro/pivot/fit/closing`, `resume_berufserfahrung/ausbildung/projekte`, `skills_block` (mechanically assembled from **Skill Group** pool ‚Äî ADR-0025). Listing-invariant content in **Facts**. _Avoid_: cv document (`cv_template.tex` is the document).

**CV Slot Contract**: In-process module `application_pipeline.cv_slot_contract` owning the canonical CV Slot-Map vocabulary and behavior-relevant projections. It exposes the exact thirteen CV Slot-Map slots, the four Cover Paragraph Pattern slots (`cover_intro`, `cover_pivot`, `cover_fit`, `cover_closing`), and marker forms such as uppercase `<<SLOT_NAME>>` where callers need LaTeX template placeholders. It is not a persisted artefact, not an adapter/port, and not permission to rename slots or change `cv.tex`, `cv_template.tex`, Cover Paragraph Pattern, Agent Skill, or CLI formats. _Avoid_: duplicated private slot constants, parser-private slot vocabulary, external seam.

**CV Skeleton**: Structure-only slot map at `<settings-dir>/cv-template/cv_skeleton.tex` (ADR-0023, relocated ADR-0035). Package-shipped, refreshable. `/write-cv` reads it as the canonical slot list and slot order, not as content guidance. _Avoid_: cv template (overloaded with `cv_template.tex`).

**CV Template Contract**: `cv_template.tex` must compile and preserve the CV Slot Contract's declared template markers so the slot-map content from `<app_dir>/cv.tex` lands in the rendered document. Visual layout is intentionally not part of the contract. _Avoid_: layout snapshot, pixel-perfect rendering.

**Compile CV Workflow**: Deep implementation behind `application-pipeline compile-cv <app_dir>` that owns config preflight, `<app_dir>/cv.tex` CV Slot-Map parsing, package LaTeX staging into `<app_dir>/.build/`, slot substitution into staged `cv.tex`, cover/resume/combined build modes, two-pass execution, pdflatex error extraction, PDF publishing, success cleanup, and failure build-dir retention. Caller-facing interface remains `compile_cv(app_dir)` / CLI only; no config or path override knobs. _Avoid_: compile-cv script, PDF writer, exposing build order to callers.

**Pdflatex Adapter**: Internal compile-cv port for the true external `pdflatex` dependency. Production preserves the existing `pdflatex` command shape, `.build/` cwd, `capture_output`, and `TEXINPUTS=".<pathsep>"` vendored-moderncv isolation; tests use a fake adapter at this port instead of patching global subprocess state. _Avoid_: subprocess seam, public compile option, TeX engine config.

**Cover Strategy**: Default writing arc for cover-letter prose: opener carries a personal, listing-specific resonance hook; middle paragraphs develop one dominant capability arc from personal history, with at most two evidence anchors extending the same argument. Projects such as Octofox, pycastle, or application-pipeline serve as evidence, not as a catalogue. `/analyse-listing` may preserve multiple hooks but should surface one lead hook for `/write-cv`; reserve hooks can feed resume, skills, or later iteration. Exact anchor depends on the analysis extracted in `/analyse-listing` and the listing. Multiple projects should not be name-dropped in the opener; additional projects can be discovered in the resume. _Avoid_: fixed cover template, project list.

**Cover Spacing**: The cover letter uses one proportional gap rule for both the opening-to-intro transition and the closing-to-sign-off transition. Both gaps track the selected cover stretch level instead of using fixed absolute spacing. _Avoid_: hardcoded gap size, asymmetric cover spacing.

**Pull-Fit**: The cover-letter argument that a listing is specifically attractive to the candidate, with capability fit implied through the same facts. _Avoid_: suitability pitch, generic motivation.

**Cover Paragraph Pattern**: A reusable slot-specific cover-letter paragraph model in `cover-patterns.md` with metadata about when to use it and one candidate-approved paragraph text that abstracts company-specific details with placeholders such as Musterfirma or Musterfrau. `/write-cv` may reuse it verbatim or optionally adapt it lightly after interactive confirmation. New patterns represent new slot-purpose plus argument-type combinations, not company or technology substitutions. Replaces loose positive exemplars for `/write-cv`. _Avoid_: generic style exemplar, free-prose inspiration, old-application archive.

**Cover Paragraph Pattern Library**: In-process `application_pipeline.cover_patterns` interface for loading or parsing `cover-patterns.md` into an owned library value and projecting caller-relevant pattern views, including all patterns and patterns for a cover slot in authored order. It owns metadata, slot, placeholder, one-paragraph, and sentence-count validation and raises `CoverPatternError` at the module seam. Missing or empty artifacts produce an empty library. It is not a port/adapter and does not change the `cover-patterns.md` format or `/write-cv` interactive choices. _Avoid_: raw pattern list, markdown parser seam, cover-patterns adapter.

**Cover Placeholder**: Controlled placeholder used inside a **Cover Paragraph Pattern** and filled only from clear `/analyse-listing` signals or candidate/project context. Allowed examples include `Musterfirma`, `Musterteam`, `Musterrolle`, `Musterprodukt`, `Musterdom√§ne`, `Mustertechnologie`, and `Musterprojekt`; `Musterprojekt` means the candidate's evidence, while `Musterprodukt` means the employer's product or platform. Skill, adjective, or work-style placeholders such as `Musterskill` or `Musterarbeitsweise` are avoided because they invite abstract capability claims. _Avoid_: free skill slot, inferred trait.

**Interactive Cover Drafting**: `/write-cv` flow where each cover paragraph is matched against **Cover Paragraph Patterns** and confirmed or rejected by the candidate. Confirmed existing patterns are written only to `cv.tex`; `cover-patterns.md` grows only when the candidate enters the main paragraph drafting loop that creates a significant new approved paragraph or explicitly asks to save it. Cover-shortening iterations after compilation affect only `cv.tex`, not `cover-patterns.md`. _Avoid_: one-shot cover generation, post-build pattern learning.

**Interactive Cover Shortening**: Post-build `/write-cv` loop for overlong cover PDFs where the LLM proposes one best shortened full-paragraph variant per cover slot, the candidate chooses one, and only that slot body in `cv.tex` changes before rebuilding. _Avoid_: fixed strip-down order for cover prose.

**Facts**: Listing-invariant data at `<settings-dir>/user-info/cv/facts.tex` (ADR-0023, ADR-0024, ADR-0027). Defines `\myFirstname`, `\myFamilyname`, `\myCity` plus `\PersonalInfo`, `\Languages`, `\Hobbies` presentation macros. `cv_template.tex` `\input`s via `\CvDataDir/facts`. _Avoid_: identity, contact (both retired filenames); per-field raw defs retired (folded into `\PersonalInfo`).

**Content Pool**: CV item macros at `<settings-dir>/user-info/cv/content_pool.tex` (ADR-0024). Each `\newcommand{\itemFoo}{...}` selected into `resume_*` slots. Per-item metadata: `always:`, `group:`, `relevance:`. Sections from `% ===== <name> =====` block headers. _Avoid_: item pool, CV pool.

**Skill**: Hard-skill item from `skills.md` in **Triage Profile** dir (#615, relocated from `search-terms/`). Dual-consumed through **TriageSkills** (ADR-0025): the **Match Judge** receives flat attribute-stripped bullet text, while `/write-cv` receives the full **Skill Group** structure for `skills_block`. Judge-only ‚Äî **Domain Pre-Filter** no longer reads it (ADR-0013). Optional `{always}` attribute (within-group floor). _Avoid_: keyword (when matching).

**Skill Group**: H2 heading in `skills.md` (ADR-0025) ‚Äî also `\cvitem{<group>}{...}` label. Unit of LLM selection for `/write-cv`: always-groups render; others picked by relevance. File order = render order. Empty groups collapse silently. _Avoid_: skill category, section (overloaded).

**Agent Skills**: Tool-consumed agent workflows seeded by `init` (ADR-0035, ADR-0048, ADR-0050). `init` no longer seeds `application-pipeline/agent-skills/`; workflow bodies are inlined directly into `.claude/skills/<workflow>/SKILL.md` and `.codex/skills/<workflow>/SKILL.md`, while tool-local support docs are seeded in `<tool-root>/_shared/` and linked via `../_shared/...`. Normal init seeds missing package-owned skill and `_shared` files; refresh overwrites those package-owned files and preserves user-added dirs and legacy `application-pipeline/agent-skills/`. `application-pipeline/skills/` remains retired legacy path. _Avoid_: subagents, prompts (overloaded).

**Keyword**: Search term from `SearchTerms.KEYWORDS` (ADR-0021). Distinct from **Skill** and **Negative Keyword**. _Avoid_: skill (when querying).

**Negative Keyword**: Entry in `SearchTerms.NEGATIVE_KEYWORDS` (ADR-0021). Case-insensitive substring match in title only (ADR-0013). No length validation, no rescue. _Avoid_: blacklist, exclusion.

**Listing ID**: Auto-increment integer, primary key in `seen.json` and `extracts.json`. Assigned at first `is_seen` miss. Next ID derived as `max(existing IDs) + 1` on store load. URLs stored as `urls: [...]` list (most-recent-first) inside the record, with a reverse index for URL-tier dedup. `PositionStub` does not carry the ID ‚Äî it is a parser-layer concept. The dedup layer assigns the ID; `RunScopedSeenResult` carries it downstream. `mark_*` methods take `listing_id: int`. _Avoid_: URL key, url id.

**Match Verdict**: Judge output per winner ‚Äî `{id, rank: 1..5}` where `id` is the **Listing ID** integer (ADR-0032). Judge prompt uses the real integer ID, not URLs. Judge ranks only; Card's Summary is from classify-time. _Avoid_: score, rating, tier (retired).

**Rank**: Integer 1..5 assigned by the judge. Not a score, not a tier. _Avoid_: tier, score, position (overloaded).

**Pool**: First-class in-process module (`application_pipeline.pool`) owning the run-scoped set of rediscovered/current-run `matched` listings keyed by **Listing ID**. Parser Intake admits `judge_pending` listings through its existing seam; Classify Stage admits newly classified `matched` listings through its existing seam. The Pool hides thread-safe **Listing ID** -> **Position Stub** storage, projects **Match Judge** candidates from **Card Store** records (**Listing ID** + **Header** + **Summary**; missing **Cards** are skipped), and applies **Match Verdicts** in **Rank** order by committing **Cards** to the **Daily Results File** before transitioning the **Deduplication Store** record to `selected_by_judge`. **Orchestrator** keeps run assembly, **Pool** ownership, **Classify Stage** startup, **Match Judge** invocation, quota sleep handling, final run summaries, and cross-run lifecycle setup; **Parser Lifecycle** owns parser execution coordination. _Avoid_: queue, candidate set - not the **Daily Top-5**.

**Daily Top-5**: ‚â§5 **Positions** the **Match Judge** returns, drawn from today's **Pool**. Each winner gets `mark_selected_by_judge(listing_id)` after Card append+fsync. _Avoid_: top-N, shortlist.

**Classify Stage**: Internal orchestration module that owns classify queue protocol, classify-ready submissions from **Parser Intake**, ADR-0047 batch accumulation, classify worker dispatch, **Quota Wall** retry, **Applied Classify Outcome** routing, **Run Metrics** classify counters, **Run Log** classify rows, and **Pool** admission for matched outcomes. Its run-internal interface lets the orchestrator start the stage, submit classify-ready **Listing ID** + **Position Stub** + **Raw Description** + parser identity, close the stage after parser threads finish, and observe completion or failure. It calls the **LLM Enricher** through the existing applied classify interface and adds no CLI, config, persistence, prompt, or external adapter seam. _Avoid_: classifier (LLM call only), LLM Enricher, classify queue (when the owning module is meant), Match Judge.

**Relevance Classifier**: Single-check LLM gatekeeper (#615, supersedes ADR-0034 three-check): domain fit + hard exclusions from `{GATE_CRITERIA}`. No candidate profile, no skill-floor check ‚Äî stretch/experience judgment deferred to **Match Judge**. Emits **Header** + **Summary** on pass. The **LLM Extractor** port remains `classify_relevance(items) -> list[RelevanceVerdict | None]`; output `<verdict id="N">{...}</verdict>` per item. Up to `claude_classify_batch_size` (default 10) listings per `claude -p` call (ADR-0046, supersedes ADR-0028). The **Classify Stage** owns ADR-0047 batch accumulation plus dispatch to the parallel worker pool (ADR-0031, ADR-0047); the **LLM Enricher** owns classifier side effects once a batch item is invoked through the applied classify interface. Unparseable verdicts become retryable applied outcomes: listing unmarked, re-discovered next run. Combined prompt via stdin (ADR-0029). Runs only on candidates surviving all non-LLM gates. _Avoid_: filter, gate.

**LLM Enricher**: Owns the classify LLM call and applied classify outcome semantics. Called by the **Classify Stage** with `(listing_id, stub, body)`, it invokes the **LLM Extractor**'s `classify_relevance`, stashes malformed batch output, runs post-LLM **Freshness Gate** arm, writes Card on matched outcomes keyed by **Listing ID**, and applies **Deduplication Store** transitions for `matched`, `out_of_domain`, and post-LLM `expired`. Returns an applied classify outcome containing only caller-needed data: matched listings for **Pool** admission, item states/counts for orchestration metrics, and observable classify event data. Unparseable per-listing verdicts remain unmarked for retry. No httpx client, no body strip, no quota sleeping. Redirect-following lives in `parsers/body_fetch.py`. _Avoid_: enricher (unqualified), extractor.

**Applied Classify Outcome**: Structured result returned by the **LLM Enricher** after raw `RelevanceVerdict | None` values have been interpreted and local side effects applied. Item states include matched, rejected/out-of-domain, retryable unparseable, and post-LLM stale/expired. Carries matched `(Listing ID, PositionStub)` data for **Pool** admission and metrics/logging facts, but does not expose raw verdict handling to the classify worker. _Avoid_: verdict (when side effects have already been applied), classify result (ambiguous).

**Freshness Gate**: Drops temporally invalid candidates (ADR-0018, ADR-0032, ADR-0038). `admit(stub, *, gate_arm, deadline=None) -> bool` at three sites: post-discover, post-enrich, post-LLM. Drops when `posted_date` exceeds `MAX_LISTING_AGE_DAYS` or `deadline < anchored_today`. `None` = no signal. Writes `expired`; on `matched ‚Üí expired` deletes extract. Parser-thread drops summed into one `freshness` counter (ADR-0043). _Avoid_: staleness filter, expiry gate.

**Content Gate**: Drops empty or placeholder body post-enrich (ADR-0030, ADR-0042). Minimum 100 chars. No `seen.json` mark ‚Äî retried next run. `admit(stripped_body, stub) -> bool`. Reason enum `{passed, empty_body, too_short}`. Effective customer: non-native-enrich parsers (ADR-0040). _Avoid_: empty-body filter.

**Domain Pre-Filter**: Title-only blacklist (ADR-0013). Substring match on **Negative Keywords**, case-insensitive. `admit(stub) -> bool`. Drops write `out_of_domain`. Log component `pipeline_prefilter`. _Avoid_: filter, gate, classifier.

**Gates Bundle**: _Retired as single call site_ per ADR-0042. Non-LLM gates invoked individually by parser thread. Pre-enrich: Freshness ‚Üí Dedup ‚Üí Pre-Filter. Post-enrich: Freshness ‚Üí Content Gate. _Avoid_: gate runner, filter chain.

**EnrichResult**: From `Parser.enrich(stub)` (ADR-0038). Carries updated `stub`, `body: str`, `mode: Literal["native", "fallback"]`. Native-enrich parsers always `"native"`; `"fallback"` = scrape-is-primary (ADR-0040). On failure, `EnrichFailedError` ‚Äî stub skipped, no `seen.json` write (ADR-0039). _Avoid_: enriched stub, enrich payload.

**Quota Wall**: Shared coordination for the parallel classify pool (ADR-0031). `raise_wall(reset_time)`, `wait_if_blocked()`, `is_active()`. `threading.Condition` + `reset_time`. One `event=quota_sleep` row per wall raise. _Avoid_: rate limiter, barrier.

**Match Judge**: Picks **Daily Top-5** from **Pool**. Single `judge_top_n(candidates)` per run. Takes `list[JudgeCandidate]` (**Listing ID** + Header + Summary), returns ‚â§5 `{id, rank}` where `id` is the integer **Listing ID**. Consumes `{CANDIDATE_PROFILE}` + `{SKILLS}` (#615). No gate criteria ‚Äî listings already passed the classifier. On non-quota error ‚Üí Failure Report, no daily file. _Avoid_: scorer.

### Deduplication and run state

**Deduplication**: Four-tier (ADR-0044): in-run `run_hit` (absorbs Cartesian overlap) plus persistent **Deduplication Store** with URL-tier (reverse index `{url ‚Üí Listing ID}`), exact-tuple-tier `(company_lc, title_lc, location_lc)`, and fuzzy-tuple-tier (token-subset, min 4 tokens, shorter ‚äÇ longer, gender markers stripped). Tuple/fuzzy fire only when all three fields non-`None`. Tuple/fuzzy hits on existing records append the new URL to the record's `urls` list (no alias records). `is_seen` writes in-memory `pending` entry on miss, assigns **Listing ID** (populates indexes immediately, no persist). Checked at two points: post-discover and post-enrich (backfilled fields). `is_seen` returns `RunScopedSeenResult` carrying `listing_id: int`: `url_hit`/`tuple_hit`/`fuzzy_hit` skip; `judge_pending` routes to Pool (includes tuple/fuzzy hits on `matched` entries ‚Äî first in run updates URL/title); `run_hit` skips within-run repeats; `miss` processes. Freshness-dropped listings: the **Freshness Gate** call owned by **Parser Intake** calls `mark_expired(stub)` to populate indexes.

**Deduplication Store Match Decision**: Private in-process policy vocabulary inside the **Deduplication Store** that maps a match fact (**URL-tier**, exact tuple tier, fuzzy tuple tier, or pending/post-enrich refresh) plus record status, cooldown age, and run-scope claim state to one `RunScopedSeenResult` and one mutation plan. Mutation plans are implementation-local and cover persisted alias writes, in-memory matched re-entry URL prepends, pending claim/refresh, run-scope claim, hit logging, and no-op outcomes. It is not a port/adapter, does not change `seen.json`, and is not exposed to **Parser Intake**, **LLM Enricher**, **Freshness Gate**, **Pool**, or callers. _Avoid_: dedup policy adapter, external decision service, new seam.

**Dedup status enum** (ADR-0014/ADR-0034/ADR-0044):
- `out_of_domain` ‚Äî Pre-Filter or Classifier rejection. Terminal-skip.
- `matched` ‚Äî classifier `matches: true`, extract written. In the **Pool**. Tuple/fuzzy hit returns `judge_pending` (first in run) or `run_hit`.
- `selected_by_judge` ‚Äî judge picked, Card appended+fsynced. Extract deleted. Suppresses within `DEDUP_COOLDOWN_DAYS` (default 30); after cooldown decays to `judge_pending` (re-enters Pool, updates URL/title).
- `expired` ‚Äî Freshness Gate drop or `matched ‚Üí expired`. Deletes extract on `matched ‚Üí expired`. Suppresses within `DEDUP_COOLDOWN_DAYS`; after cooldown decays to `miss` (re-enters pipeline). `status_last_changed` refreshed on each freshness re-drop.
- `pending` ‚Äî in-memory only, not persisted. Claimed by first `is_seen` miss; overwritten by classify worker's `mark_*`.
- ~~`enrich_failed`~~ ‚Äî _retired_ (ADR-0039). URLs stay unrecorded, retried next run.
- ~~`external_redirect`~~ ‚Äî _retired_ (ADR-0032). Redirects followed silently.

**Error semantics for `mark_*`** (single-writer Pi):
- **Pre-filter drop** ‚Üí `mark_out_of_domain(stub)`. No LLM cost.
- **Classifier non-quota error** ‚Üí listing NOT marked; orchestrator continues; retry next tick. Malformed stashed to `failures/malformed/`.
- **Classifier `ClaudeUsageLimitError`** ‚Üí **Quota Wall** raises; workers sleep until reset+2min. Cron-anchored day handles midnight.
- **Judge non-quota error** ‚Üí no daily file, Failure Report, Pool intact.
- **Body fetch failure** ‚Üí stub skipped, URL unrecorded (ADR-0039). Native-enrich: 401/403/5xx/3xx/JSON decode ‚Üí Failure Report + parser dead; 404/400/422/retries-exhausted ‚Üí silent skip.
- **Oversized/malformed LLM output** ‚Üí stashed, no `seen.json` write, retried next run.

State at `<settings-dir>/.runtime-data/seen.json` (ADR-0037; synced via Syncthing, ADR-0002). Shape: `{listing_id: {urls: [...], company_lc, title_lc, location_lc, status, status_last_changed}}` ‚Äî keyed by **Listing ID** integer. `urls` list ordered most-recent-first; no alias records. On-load migration from legacy URL-keyed format (collapses alias chains into `urls` lists, assigns integer IDs). `DeduplicationStore` exposes four methods taking `listing_id: int`: `mark_out_of_domain`, `mark_matched`, `mark_selected_by_judge`, `mark_expired`. Single-writer (ADR-0002); internal lock covers concurrent classify/judge writes. Config: `DEDUP_COOLDOWN_DAYS: int = 30` (ADR-0044). _Avoid_: duplicate filtering, URL filtering.

### Display

**Card**: Fixed markdown block per winner (ADR-0033): `# **{rank}:** {title}` (blank line) metadata lines + URL (blank line) `{Summary}` (blank line) `---` (blank line) `{Raw Description}` (blank line) `---`. _Avoid_: card template, headline.

**DailyResultsFile**: Module that owns both rendering and durability for the **Daily Results File** artefact. Card shape is hardcoded inside this module per ADR-0033. Write durability is `write + flush + fsync` per ADR-0015. Public interface: `ensure_initialized()` (mkdir parent dir) and `commit(*, rank, header, summary, url, body)` (render Card then append to file). `OSError` is wrapped as `ResultsFileError` at the public interface edge. Path bound at construction. _Avoid_: renderer, formatter, results file manager, writer, output manager.

**Atomic Write Helper**: `write_atomic(path, payload: bytes)` ‚Äî crash-safe atomic overwrite via `.tmp` sibling + `os.write` ‚Üí `os.fsync` ‚Üí `os.replace`. Used by **Deduplication Store** and **Extract Store**. _Avoid_: persistence helper, file writer (overloaded with **DailyResultsFile**'s append).

### Observability

**Log Artifacts**: Under `<settings-dir>/.runtime-data/logs/` (ADR-0037), laid out by reader (ADR-0012), layer-subdirs (ADR-0036). Four types: `<layer>/<comp>.events.jsonl`, `lifecycle.jsonl`, `run.log`, `<layer>/<comp>.transcripts.jsonl`. Component identifiers stay layer-prefixed (`parser_`, `llm_`, `pipeline_`) in all data ‚Äî subdir replaces prefix only on the filename. `llm_classify_relevance` for classifier, `llm_judge_match` for judge. _Avoid_: log file (unqualified).

**Run Log**: Per-run instance writing **Log Artifacts**. Constructed once from `cfg.logs_dir`; threaded into every component. Safe to share across threads ‚Äî each method opens, writes, closes per call. _Avoid_: log writer, logger (overloaded).

**Parser Lifecycle**: Internal parser-thread orchestration module that owns parser-thread startup, per-query discovery loops, `query_started` / `query_ended` **Log Artifact** rows, parser lifecycle control messages, not-served query accounting, parser-dead **Failure Report** recording, stall watchdog logging and stack capture, parser thread joins, and parser summary emission. It accepts entered **Parser** adapters plus existing run collaborators from the **Orchestrator** and calls **Parser Intake** for each discovered **Position Stub**. Its interface hides queue sentinels, thread state records, watchdog bookkeeping, and parser timing details; it returns only caller-relevant lifecycle observations. It does not own **Parser Intake** gates, **Classify Stage** batching, **Pool** ownership, **Match Judge** flow, Config loading, parser registry resolution, or external time/thread ports. _Avoid_: parser thread helper, parser queue protocol, lifecycle adapter.

**Maintenance**: Post-run cleanup behind `run_maintenance(logs_dir, failures_dir)`. Owns **Log Artifact** retention recursively beneath `<settings-dir>/.runtime-data/logs/`: root shared artifacts (`run.log`, `lifecycle.jsonl`), ADR-0036 layer subdirs (`parser/`, `llm/`, `pipeline/`), and old flat artifacts left by no-migration. Applies the existing tail-line rule per file and treats per-file filesystem errors as best-effort. Also deletes old root-level **Failure Report** markdown from `<settings-dir>/.runtime-data/failures/`; it does not recurse failure cleanup or expose Log Artifact layout to callers. _Avoid_: log rotation, cleanup adapter, exposing log layout to callers.

**Status Display**: Live in-process progress view (ADR-0043). `StatusDisplay` Protocol: `RichStatusDisplay` (tty) and `PlainStatusDisplay` (cron). Uniform counters: queued/dropped/forwarded. Each parser gets parser row + gates row (non-zero drops only). `llm_classify_relevance` row adds `malformed` + `classifying` counters; `queued` shows current depth. Judge: terminal message only. _Avoid_: progress bar, TUI, dashboard.

### Sources & extraction

**Source**: A `SourceEntry` in `Config.SOURCES` carrying a **Parser Type**. Source = config entry; Parser = module. One-to-one in v1. _Avoid_: site, board.

**ParserHttp**: Per-parser HTTP layer owning pacing (0.5s), retry-with-backoff on `{429, 502, 503, 504}` (max 3), status classification, redirect handling, **Log Artifact** rows, and discover/enrich error conversion through `get()` / `enrich_get()`. True external HTTP sits behind an internal named transport port; production uses an `httpx` transport adapter for client lifetime, headers, timeout, and no-follow-redirect behavior, while tests use a scripted transport adapter. Parser modules depend on `ParserHttp`, not on raw transport adapters. _Avoid_: http client, fetcher, parser transport adapter (when the parser-facing module is meant).

**Parser**: Module in `src/application_pipeline/parsers/`. Two methods (ADR-0038): `discover(query) -> Iterable[PositionStub]` and `enrich(stub) -> EnrichResult`. Context manager owning `httpx.Client`; no shared mutable state (ADR-0004, amended ADR-0042 ‚Äî threads now do `enrich()` I/O inline). On the parser thread, `discover()` yields **Position Stubs** into **Parser Intake**, which owns the per-stub gates and handoff effects; `Parser.enrich()` still runs on that same parser thread through the **Parser** interface. `has_native_enrich = True` ‚Üí exclusive native path, no fallback (ADR-0040). `body_selector` is parser-private. In-parser dedup retired (ADR-0041). **Keyword-match invariant** (ADR-0013): every stub matches its query keyword in the title. _Avoid_: scraper, fetcher.

**Parser Intake**: Internal parser-thread handoff module that processes one **Position Stub** after `discover()` and applies the complete per-stub path before **Classify Stage** or **Pool** admission. Owns **Freshness Gate** (`discover`), **Deduplication** (post-discover), **Domain Pre-Filter**, `Parser.enrich()`, **Deduplication** (post-enrich), **Freshness Gate** (`post_enrich`), **Content Gate**, **Deduplication** counter recording, parser-row **Status Display** updates, Parser Intake skip **Log Artifact** emission, **Pool** admission for `judge_pending` rediscovery, and forwarding of classify-ready listings to the **Classify Stage**. Its public interface exposes one narrow per-stub call and does not return parser-row metric names, dedup-event tuples, log artifact payloads, **Pool** admission records, or classify handoff data. The **Parser Lifecycle** keeps parser/query lifecycle, parser-dead handling, stall watchdog behavior, thread joins, and parser summaries. The **Orchestrator** keeps run state, **Classify Stage** startup, classify worker topology, and final judge flow. `Parser.enrich()` still runs on the parser thread through the **Parser** interface. _Avoid_: gate bundle, new port, parser adapter.

**Position Stub**: Result of `discover()` ‚Äî `url, title, source, posted_date?, deadline?, company?, location?`. Required: url, title, source. Optional fields drive post-discover gates and Header pre-fills. `deadline` feeds the **Freshness Gate** `deadline` arm (`deadline < anchored_today` ‚Üí drop). `enrich()` may back-fill optional fields. _Avoid_: preview, summary.

**External Redirect**: _Retired_ per ADR-0032. Redirects followed silently in `parsers/body_fetch.py`. _Avoid_: do not reintroduce.

**Parser Type**: String on `SourceEntry` mapping to parser filename. _Avoid_: adapter.

**Location**: Sealed `City(name) | Remote` passed via `ParserQuery.location`. _Avoid_: place, geo.

**City**: `City(name)` arm. Normalized via `normalize()`. No central catalog. _Avoid_: town, place.

**Remote**: `Remote` arm. Per-parser `remote_wire()`. _Avoid_: homeoffice (when the type variant is meant).

**Location Coverage**: Per-parser Protocol ‚Äî `serves(name)`, `to_wire(name)`, `serves_remote`, `remote_wire()`. Validated at load time (ADR-0009). _Avoid_: location map, slug table.

**LLM Extractor**: Protocol: `classify_relevance(items: list[ClassifyItem]) -> (list[RelevanceVerdict | None], CallUsage)` and `judge_top_n(candidates) -> (list[MatchVerdict], CallUsage)`. `RelevanceVerdict` = `{matches, header?, summary?}` (ADR-0034). `MatchVerdict` = `{id: int, rank}` where `id` is the **Listing ID**. Production implementation: `ClaudeExtractor` via `claude -p` subprocess (ADR-0029 wire shape). Models: `haiku` classifier, `haiku --effort medium` judge (ADR-0010). Tags: `<verdict>`/`<verdicts>` via **Agent Output Protocol** (ADR-0010). _Avoid_: LLM, model (unqualified).

**Agent Output Protocol**: `extract_json_block(text, tag) -> Any` + `AgentOutputProtocolError` (ADR-0010). Finds rightmost closing tag, walks back, strips optional fence, `json.loads`. Fallback: when tags absent, attempts bare-JSON extraction from markdown code fence; logs `protocol_fallback` on recovery. _Avoid_: output parser, response handler.

**Pagination**: Page fetches until source returns empty. No dedup-driven early-stop (ADR-0011). `max_results` retired (ADR-0041). _Avoid_: paging.

### Invocation

Distributed via PyPI (ADR-0020). Install: `.venv/bin/pip install application-pipeline`, `init`, `bash setup/cron-install.sh`. Home dir hardcoded to `<cwd>/application-pipeline/` (ADR-0022).

- `application-pipeline run` reads `<cwd>/application-pipeline/config.py`.
- `application-pipeline init [--refresh]` invokes **Init Bootstrap** to seed into `<cwd>/application-pipeline/` and tool-specific wrapper roots.
- `application-pipeline compile-cv <app_dir>` compiles per-listing draft from slot-map. Windows paths normalized to POSIX for `\def\CvDataDir{...}` (ADR-0024, ADR-0023).

Fail loud-and-fast (exit 2) if `config.py` missing. Cron weekdays 00:30 (ADR-0017). Each tick: `pip install --upgrade` (√ó2), `init --refresh`, `run`. **`init --refresh`** overwrites `setup/*.sh`, `cv-template/`, tool-root inlined skill bodies, tool-root `SKILL.md` files, and package-owned tool-root `_shared` docs (ADR-0035, ADR-0048, ADR-0050), deletes `layout.py` (ADR-0033) and obsolete `skills/` dir (ADR-0035), and leaves legacy `application-pipeline/agent-skills/` untouched for manual migration cleanup. Seeds-if-missing for `config.py`, `user-info/*`, `.gitignore` (ADR-0037). Never touches `.runtime-data/`. Templates organised into routing buckets (ADR-0035). No flock ‚Äî single-writer on the Pi, overlapping ticks cannot occur. `pycastle/` in this repo is an unrelated RALPH Loop plugin used to *build* this project.

## Relationships

- A listing reaches a **Daily Results File** by passing (in order, ADR-0038/0042/0044): parser thread discovery yields a **Position Stub**; **Parser Intake** handles **Freshness Gate** (discover; drop -> `expired`), **Deduplication** (post-discover; skip/**Pool** re-entry/miss), **Domain Pre-Filter** (title; drop -> `out_of_domain`), `Parser.enrich()`, **Deduplication** (post-enrich; backfilled fields/**Pool** re-entry), **Freshness Gate** (post-enrich), and **Content Gate** (body). **Parser Intake** admits rediscovered matched listings directly to **Pool** or forwards accepted listings to the **Classify Stage**. The **Classify Stage** invokes the **LLM Enricher** for the raw **Relevance Classifier** call plus malformed stashing, post-LLM **Freshness Gate**, Card write, **Deduplication Store** transition, and matched-outcome routing into the **Pool** before the **Match Judge** picks **Daily Top-5** from the **Pool**.
- During run assembly, the **Orchestrator** enters configured **Parser** adapters and prepares run collaborators, then crosses the **Parser Lifecycle** seam for parser execution. **Parser Lifecycle** calls **Parser Intake** per **Position Stub** and hands classify-ready output to the already-started **Classify Stage** through its existing handoff.
- The **Pool** is the run-scoped **Listing ID** set admitted through **Parser Intake** `judge_pending` rediscovery and **Classify Stage** `matched` outcomes; it is implemented by `application_pipeline.pool` and derives **Match Judge** candidates from **Card Store** entries.
- The **Match Judge** runs once per run, takes Header + Summary + Candidate Profile + Skills, returns ‚â§5 `{id, rank}`.
- **Triage Profile** reaches LLM via two disjoint paths (#615): `{GATE_CRITERIA}` ‚Üí classifier only; `{CANDIDATE_PROFILE}` + `{SKILLS}` ‚Üí judge only. `NEGATIVE_KEYWORDS` reaches **Domain Pre-Filter** directly.
- **Triage Profile** prompt-slot loading is local-substitutable: tests use temporary filesystem state plus the owned in-process **TriageSkills** module, not mocks, a remote dependency, or a new port/adapter.
