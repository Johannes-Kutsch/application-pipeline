# Application Pipeline

Personal job-discovery and triage pipeline. Fetches listings from a small set of sources, classifies relevance with Claude, accumulates an in-domain **Pool** across days, emits one dated **Daily Results File** per day carrying the **Daily Top-5** **Cards** ranked by the **Match Judge**. Application authoring (CV/cover letter) is out of scope for v1.

## Scope

- **In scope (v1):** Working **Parsers** for **Bundesagentur**, **stellen.hamburg**, **jobs-beim-staat**, smoke-tested standalone on the laptop.
- **In scope (v1.1):** Full pipeline on Pi — orchestrator, **LLM Enricher** (per ADR-0041 — body fetch + strip + **Relevance Classifier** producing **Header** + **Summary**), **Match Judge** (one call per run, picks **Daily Top-5** from the **Pool**), **Deduplication**, daily file, cron once per day, Syncthing sync.
- **Out of scope:** **Profile** ingestion, additional commercial parsers, browser automation. (CV / cover-letter LaTeX compilation now in scope via `application-pipeline compile-cv` — see Invocation.)

## Language

### Pipeline artifacts

**Config**: A `config.py` at the root of the settings directory, hardcoded to `<cwd>/application-pipeline/` (ADR-0029) controlling pipeline shape — `SOURCES`, `LOCATIONS`, `INCLUDE_REMOTE`, Claude settings (optional `claude_cli_path`), `MAX_LISTING_AGE_DAYS: int` (default 180, `≥ 1`) driving the **Freshness Gate**'s `posted_date` arm, and `claude_classify_parallelism: int` (default 4, `≥ 1`) sizing the **Relevance Classifier**'s parallel worker pool (ADR-0040). The former `claude_classify_batch_size` knob is retired (ADR-0036 supersedes ADR-0014's batching). No path override knobs: `user-info/` and prompt locations are all canonical under the settings dir (ADR-0011 amended). `layout.py` retired per ADR-0042. **Does NOT** carry `KEYWORDS`/`SKILLS`/`NEGATIVE_KEYWORDS` — those live in **SearchTerms** (ADR-0028). Plain Python literals; `#` comments. Materialised on first bootstrap from `src/application_pipeline/templates/config.py` via `application-pipeline init <dir>` (ADR-0011); the user edits in place from then on. Never overwritten by `init --refresh` (ADR-0011, refined: refresh overwrites `setup/` only). Loaded by **Config Loader**, returns a frozen typed `Config`. Each `SourceEntry` carries its own `max_results` (default 1000). Cron schedule lives in crontab, not Config (ADR-0024, once daily at 00:30 local). _Avoid_: config file, settings file, search config.

**SearchTerms**: The user-authored search/filter knobs lifted out of **Config** per ADR-0028 — `KEYWORDS`, `SKILLS`, `NEGATIVE_KEYWORDS`. Lives as three markdown files under `<settings-dir>/user-info/search-terms/` — `keywords.md`, `skills.md`, `negative-keywords.md` (ADR-0032; supersedes the prior single-file layout). Filename *is* the section — no `##` header inside each file. Body shape: flat `-` bullet entries; `keywords.md` and `negative-keywords.md` may carry topical sub-labels as plain non-bullet lines, and `skills.md` additionally carries H2 **Skill Group** headings with pandoc-style `{...}` attributes (ADR-0033). Loaded by a dedicated loader (one flat-bullet parse per file — headings and attribute blocks in `skills.md` are ignored at the pipeline parse layer), exposed as a frozen `SearchTerms` object threaded alongside `Config` into the orchestrator, **Domain Pre-Filter**, and `ClaudeExtractor` construction. `keywords.md` missing OR present-but-zero-bullets raises `SearchTermsError` (degenerate empty pipeline); `skills.md` and `negative-keywords.md` are optional — missing file treated as empty list. No length validation on negative keywords (lifted in ADR-0028; was `≥ 3` when on Config). Distinct from the **Triage Profile** — they share the `user-info/` tree but live in different sub-dirs and have different downstream consumers (SearchTerms drives parser orchestration / prefilter / judge's `{skills}` slot; Triage Profile is LLM-prompt content via `{USER_INFO}`). Hard cutover: installs with the pre-ADR-0032 layout (single `search-terms.md` at user-info root) error on first run until the three files exist under `search-terms/`. _Avoid_: search config, term file.

**Layout**: _Retired_ per ADR-0042. The `layout.py` file, `Layout` dataclass, placeholder vocabulary, `PLACEHOLDER_GROUPS` / `CARD_TEMPLATE` constants, named-group separator logic, and `str.format_map` plumbing are all gone. Card structure is hardcoded in the **Renderer** (one-line concatenation prepending `# **{rank}:** ` to the LLM-authored Header + Summary). `init` no longer seeds `layout.py`; `init --refresh` deletes it from existing installs. `LayoutError` retired. _Avoid_: do not reintroduce as a knob — Header content is shaped by the classify prompt, not a runtime template.

**Daily Results File**: One dated markdown file per calendar day at `<settings-dir>/results/YYYY-MM-DD.md`, holding the **Daily Top-5** as **Cards** in **Rank** order. Date is **cron-anchored** (ADR-0021/0023) — a run started day X writes to day X's file even if it crosses midnight sleeping through quota. No `FILE_HEADER`, no preamble, no **Run Divider**. If today's pool had fewer than 5, the file carries however many cards; if pool was empty, no file is written. Write-once on the host; if a sync channel is configured (e.g. Syncthing), the file propagates outward read-only. _Avoid_: results file (without "daily"), `current.md` (gone), dated results file.

**Failure Report**: A markdown file at `<settings-dir>/failures/<timestamp>.md` written on a failed run — `cron.sh` pipeline-stage errors (ADR-0027; `init --refresh` or `run` exits non-zero — pip-upgrade failures warn-and-continue instead), orchestrator runtime errors, **Match Judge** failure (no daily file), and non-quota classifier errors that left the run with no writeable output. Per ADR-0010. Acknowledged by deleting the file. Quota errors do NOT trigger a Failure Report — they sleep until reset (ADR-0023). Per-listing soft failures (oversized stripped body, malformed LLM output; ADR-0041) do **not** trigger Failure Reports either — they stash to sibling directories `<settings-dir>/failures/oversized/` and `<settings-dir>/failures/malformed/` with no `.seen.json` write, and the URL is retried next run. _Avoid_: incident report, error log.

**Position**: A single job listing surviving the **Relevance Classifier** and the **Match Judge**'s top-5 selection. Identified within its **Daily Results File** by its rendered **Card**. The Card shape is fixed (ADR-0042): `# **{rank}:** {Header}` followed by `{Summary}`. _Avoid_: job, listing, vacancy.

**Position Schema**: _Retired_ per ADR-0041. Parsers no longer construct a `Position` — `enrich()` retires; body fetch and field extraction move to the shared **LLM Enricher**. Only `PositionStub` survives in `parsers/types.py`. The closed-enum fields (`salary`, `contract_type`, `employment_type`, `work_model`, `deadline`) are no longer extracted as separate fields — their content lives inside the LLM-authored **Header** string. The "parsers never guess" invariant retires with this dataclass; the **LLM Enricher** is allowed to infer from body text (capable-model regime, 2026-05-22 grilling). _Avoid_: do not reintroduce per-field extraction in parsers.

**Raw Description**: The full body text of a candidate listing, fetched and stripped by the **LLM Enricher** (per-source CSS selector when available via `Parser.body_selector`, generic library fallback — `trafilatura` / `readability-lxml` — for redirect destinations and selectorless sources). Per ADR-0041, raw_description is **fed to the LLM as classify input but never persisted and never rendered into a Card**. Empty stripped body is dropped by the **Content Gate** (ADR-0039) before the LLM call; the URL is not marked in `.seen.json` so the next run re-fetches. Over-cap stripped bodies are dropped, the raw HTML is stashed to `<settings-dir>/failures/oversized/<source>-<url-slug>.html`, and `.seen.json` is not marked (re-check next run, same pattern as Content Gate). _Avoid_: description (when full text is meant); not to be confused with the LLM-authored **Header** / **Summary**.

**Structured Extract**: _Retired_ per ADR-0041 (which supersedes ADR-0022). Replaced by **Header** + **Summary** — both LLM-authored strings produced by the classifier alongside `in_domain: true`. `extracts.json` is `{stable_id: {header, summary}}`. Wipe-state migration per ADR-0024 — no auto-translation from the prior shape. _Avoid_: do not reintroduce; field-level extraction is gone.

**Header**: The three-line block authored by the **LLM Enricher** at classify time (ADR-0041) — title on line 1, `company · location · work_model` on line 2, `posted_date · seniority · salary` on line 3. The LLM substitutes a known value when the parser pre-filled a `PositionStub` field, infers from the body when it can, or drops the segment (and its `·` separator) when neither parser nor body offers a signal. Persisted as a single string in `extracts.json`. Rendered into the **Card** verbatim under the rank prefix. _Avoid_: card top, headline.

**Summary**: The prose paragraph authored by the **LLM Enricher** at classify time (ADR-0041), describing the role in the **Triage Profile**'s frame. Persisted alongside **Header** in `extracts.json`. Rendered into the **Card** below the Header. The **Match Judge** ranks on Header + Summary directly — no structured comparison fields. _Avoid_: match verdict summary (retired with `MatchVerdict.summary` per ADR-0041), description (overloaded with **Raw Description**).

### Filtering & scoring

**Triage Profile**: The applicant's self-description plus rules deciding in-domain / good-match. Lives in `<settings-dir>/user-info/triage-profile/` (ADR-0032; previously at `user-info/` root) as four markdown files — `self-description.md` (fed into both LLM call sites), `domain-fit.md` (classifier), `match-criteria.md` (judge), and `writing-style.md` (PRD-#16-v2.1 authoring skills only, NOT injected into v1 prompts). Per ADR-0016, the **Prompt Loader** concatenates the three v1-consumed files per call site; the **LLM Extractor** injects the payload into the hardcoded prompt's `{USER_INFO}` slot wrapped in `<user-info>` tags. Bullets/keywords, German, "extremely concise, sacrifice grammar for concision" (ADR-0026). Reused as v2 authoring context — no separate CV Profile. _Avoid_: profile (without qualifier), bio, CV Profile (retired before being built — do not reintroduce).

### CV authoring

**CV Slot-Map**: The per-listing file `<app_dir>/cv.tex` emitted by the `/write-cv` skill and consumed by `compile-cv` (ADR-0030). Sectioned-marker syntax: `^%% SLOT: <name>$` introduces a body running until the next `%% SLOT:` line or EOF. Bodies are raw TeX fragments (multi-line German prose with `\href`, `\textit`, umlauts — no escape rules). Thirteen committed slots: `recipient_company/name/street/zip_city` (DIN 5008 Anschriftenfeld, four lines — per issue #472), `opening`, `cover_intro/pivot/fit/closing`, `resume_berufserfahrung/ausbildung/projekte` (per issue #453 — supersedes single `<<RESUME_BODY>>`), `skills_block` (mechanically assembled by `/write-cv` from the **Skill Group** pool in `skills.md`, not freehand-authored — ADR-0033; the slot body carries the rendered `\cvitem` rows verbatim so `cv.tex` remains the self-contained artifact). Everything else (name/address/photo/languages/hobbies/title) is listing-invariant and lives in **Facts**. _Avoid_: cv document (CV Slot-Map is the *input*, not the document; `cv_template.tex` is the document).

**CV Skeleton**: The format-by-example file at `<settings-dir>/skills/cv_skeleton.tex` (ADR-0030). Package-shipped via `src/application_pipeline/templates/skills/cv_skeleton.tex`, seeded by `init`, **refreshable** by `init --refresh` (ADR-0011 amendment — second refreshable item alongside `setup/*.sh`). Carries the slot list + per-slot prompt guidance. The `/write-cv` skill reads the skeleton from the settings dir (never from `src/`) and injects it into the prompt as the output-format example. The skill's interface contract is the settings dir; the skill stays agnostic of install location. _Avoid_: cv template (overloaded with `cv_template.tex` — the skeleton is a *prompt example*, not the LaTeX document).

**Facts**: The listing-invariant data file at `<settings-dir>/user-info/cv/facts.tex` (ADR-0030; relocated under `cv/` per ADR-0032; restructured per ADR-0035). Defines `\myFirstname`, `\myFamilyname`, `\myCity` (raw scalars used by the template across cover + resume), plus three presentation macros — `\PersonalInfo`, `\Languages`, `\Hobbies` — each a block of `\cvitem` / `\cvlistitem` calls the user freely edits to add or drop rows. `cv_template.tex` `\input`s facts via `\CvDataDir/facts` and calls `\firstname{\myFirstname}`, `\familyname{\myFamilyname}`, `\title{Lebenslauf}`, `\photo[120pt][1pt]{\CvDataDir/profile}`; the resume's Persönliche Informationen section emits `\cvitem{Name}{...}` then `\PersonalInfo`. Replaces the previous `identity.tex` + `contact.tex` split. _Avoid_: identity (subset only), contact (subset only) — both retired as filenames; the per-field raw defs (`\myStreet`, `\myZip`, `\myPhone`, `\myEmail`, `\myGithub`, `\myLinkedin`) and the `\Xdisplay` mirror defs are also retired (folded into `\PersonalInfo`).

**Content Pool**: The pool of CV item macros at `<settings-dir>/user-info/cv/content_pool.tex` (relocated under `cv/` per ADR-0032; `cv_template.tex` `\input`s it via `\CvDataDir/content_pool`). Each `\newcommand{\itemFoo}{...}` is a `\cventry`/`\subcventry` body the `/write-cv` skill can select into a `resume_*` slot. Per-item metadata header (ADR-0030) is three fields: `always:` (boolean — must appear in every application), `group:` (string — mutually exclusive variants), `relevance:` (`<jobtype>=<high|medium|low>, ...`). Sections (`Berufserfahrung` / `Ausbildung` / `Projekte`) are derived from `% ===== <name> =====` block headers above each item, not stored per-item. The previous `section:`, `tags:`, `summary:` fields are retired (section was duplicate of position; tags overlapped with relevance; summary drifted from `\cventry` body — Claude reads bodies directly). _Avoid_: item pool, CV pool, content database.

**Skill**: A hard-skill or technology item from **SearchTerms**' `SKILLS` (ADR-0028; previously on Config). Dual-consumed (ADR-0033): the pipeline harvests every bullet body in `skills.md` as a flat list, rendered into the **Match Judge** prompt's `{skills}` slot at `ClaudeExtractor` construction; `/write-cv` reads the same file with its full **Skill Group** + attribute structure to assemble the CV's `skills_block` slot. Per ADR-0019, `SKILLS` is **only** consumed by the judge prompt — the **Domain Pre-Filter** no longer reads it. The LLM's `matched`/`missing` lists are open-vocabulary. An item carries an optional `{always}` attribute (within-group render floor; does **not** promote its group into the CV — ADR-0033). _Avoid_: keyword (when matching).

**Skill Group**: An H2 heading in `skills.md` (ADR-0033) — also the `\cvitem{<group>}{...}` category label in the rendered CV. The unit of LLM selection for `/write-cv`: always-groups (`{always}`) always render; non-always groups are picked by the LLM against the listing's jobtype via per-group `<jobtype>=<high|medium|low>` relevance entries. Within an included group every `{always}` **Skill** renders plus zero-or-more LLM-picked items; excluded groups contribute nothing. File order = render order; the LLM does not reorder groups or items. Empty included groups (zero `always`, zero picks) collapse silently — no empty `\cvitem` row. _Avoid_: skill category (the prior judge `{matched}`/`{missing}` open-vocab lists are retired per ADR-0041, so no longer a clash — but the term still overloads), section (overloaded with **Content Pool**'s `% ===== <name> =====` block headers).

**Keyword**: A search term used to query a **Source**, from `SearchTerms.KEYWORDS` (ADR-0028; previously `Config.KEYWORDS`). Distinct from a **Skill** (does not affect judgment) and from `NEGATIVE_KEYWORDS` (which drives the **Domain Pre-Filter**). _Avoid_: skill (when querying), negative keyword (when querying).

**Negative Keyword**: An entry in `SearchTerms.NEGATIVE_KEYWORDS` (ADR-0028; previously on Config). Patterns that, if found via case-insensitive substring match in a **Position**'s **title** (body text not consulted, per ADR-0019), cause the **Domain Pre-Filter** to drop the listing. No length validation (lifted in ADR-0028). No rescue mechanism. _Avoid_: blacklist, exclusion.

**Match Verdict**: The structured output of the **Match Judge** for each of the 5 winners. Per ADR-0041, simplified to `{id, rank: 1..5}` — the Judge ranks only. The Card's Summary text is the classify-time **Summary**, not a Judge-produced summary; `matched` / `missing` lists retire (rank-signal is implicit in the Header/Summary the Judge reads). _Avoid_: score, rating, tier (retired).

**Rank**: The `rank` field — integer 1..5 assigned by the judge when picking the **Daily Top-5**. Surfaced in the **Card** via `{rank}` placeholder. Not a score (no cross-day comparability); not a tier (no enum). _Avoid_: tier (retired), score, position (overloaded).

**Pool**: The implicit set of in-domain **Positions** eligible for today's **Match Judge** call — every URL in `.seen.json` with `status == in_domain` re-discovered by a parser in the current run. No separate data structure; computed per-run as `{url ∈ enriched_today : .seen.json[url].status == in_domain}`. Enter: classified `in_domain` (status flips, extract written). Exit: judge picks it (status → `selected_by_judge`, extract deleted) or no longer re-discovered (record sticks but never re-enters). No pool size cap, no TTL eviction in v1 (ADR-0022). _Avoid_: queue, candidate set, judge list — and not to be confused with **Daily Top-5** (the pool is the input set; the top-5 is the selected output).

**Daily Top-5**: The 5 **Positions** the **Match Judge** returns from its single end-of-run call, drawn from today's **Pool**. ≤5 if pool was smaller. Surfaced as **Cards** in **Rank** order. Each winner gets `mark_selected_by_judge(stub)` after its **Card** is appended+fsynced, removing it from the **Pool**. _Avoid_: top-N, shortlist.

**Relevance Classifier**: The classify-time LLM call site, subsumed into the shared **LLM Enricher** per ADR-0041. Decides whether a candidate listing is in the applicant's professional domain (AI / Data / Game Dev / SWE) and, when in-domain, also emits the rendered **Header** and **Summary** for the Card. Implemented as `LLMExtractor.classify_relevance(item: ClassifyItem) -> (RelevanceVerdict, CallUsage)`; output shape `<verdict>{"in_domain": true, "header": "...", "summary": "..."}</verdict>` or `<verdict>{"in_domain": false}</verdict>`. One `claude -p` subprocess per listing (per ADR-0036, amended by ADR-0041 — only the payload shape changes). The parallel worker pool (N = `Config.claude_classify_parallelism`, default 4 per ADR-0040) and shared **Quota Wall** are unchanged. The full prompt (instructions + `{USER_INFO}` + per-listing Header pre-fills + stripped body) is sent as a single combined message via stdin (ADR-0038). The prompt instructs the model to short-circuit on the first strong out-of-domain signal — precision-favoring trade. On `ClaudeUsageLimitError` the worker publishes `reset_time` to the **Quota Wall** (ADR-0023, ADR-0040). Malformed response (bad JSON, missing tag, empty `header` or `summary` when in-domain) fails that one item — no `.seen.json` write, raw output stashed to `<settings-dir>/failures/malformed/<source>-<url-slug>.txt` per ADR-0041, retried next cron tick. Every call writes event + transcript rows to `llm_classify_relevance.events.jsonl` / `.transcripts.jsonl`. Runs only on candidates that survive the **Domain Pre-Filter**, the pre-LLM arm of the **Freshness Gate**, body fetch, body strip, the **Content Gate**, and have no prior `.seen.json` record. Off-domain candidates get `mark_out_of_domain(stub)` immediately. _Avoid_: filter (ambiguous — see **Domain Pre-Filter**), gate.

**LLM Enricher**: The shared pipeline stage that replaced per-source `enrich()` per ADR-0041. Owns: generic HTTP fetch with silent redirect-following (no per-parser host classification, ADR-0013 / ADR-0037 retired), body strip via per-source `Parser.body_selector` when present or generic library (`trafilatura` / `readability-lxml`) fallback, hard token cap on stripped body with raw-HTML stash to `failures/oversized/` on overflow, the **Content Gate** drop on empty stripped body, then the **Relevance Classifier** LLM call. Single LLM call per candidate produces `{in_domain, header, summary}`. Subsumes the call site that was previously split across `Parser.enrich()` + classifier. _Avoid_: enricher (without "LLM" — overloaded with `mark_in_domain`'s "enrich" verb), extractor.

**Freshness Gate**: A gate dropping temporally invalid candidates (ADR-0025, amended by ADR-0041). Runs **twice**: a pre-LLM arm (`admit_stub(stub) -> bool`) immediately after `discover()` when `PositionStub.posted_date` is present — cheap pre-LLM drop, no body fetch paid; a post-LLM arm (`admit(candidate) -> bool`) inside the **LLM Enricher** after extraction, catching `posted_date` values that only the LLM could infer. Drop conditions on both arms: `posted_date is not None and (anchored_today - posted_date).days > MAX_LISTING_AGE_DAYS`, **or** `deadline is not None and deadline < anchored_today`. `None` on a field is "no signal, don't drop on that field alone"; both `None` → pass. `anchored_today` is the cron-anchored logical date (ADR-0021). Drops write status `expired`; on `in_domain → expired` the URL's `extracts.json` entry (`{header, summary}` per ADR-0041) is deleted. Per-position transcript gains a `gate_arm: "discover" | "post_enrich"` field. Log component `pipeline_freshness`. Future-dated `posted_date` (negative age) passes silently. _Avoid_: staleness filter, expiry gate, date filter.

**Content Gate**: A gate inside the **LLM Enricher** (per ADR-0041; originally per ADR-0039) running after body strip and before the classifier LLM call, dropping candidates whose stripped body is empty. Owns its full operational protocol — verdict, per-position transcript, per-run aggregate — behind `admit(stripped_body, stub) -> bool` / `emit_run_complete()`. Drops do **not** mark `.seen.json`: the URL is re-discovered, re-fetched, and re-checked on the next run, giving the source a chance to publish the body later. Bucket-named (not `EmptyBodyGate`) so future content-validity rules — minimum body length, body-language detection — extend the same module without a rename. Reason enum is open-ended `{passed, empty_body}`. Log component `pipeline_content`; transcript record `url, title, source, passes, reason, body_len`; per-run aggregate `content_considered, content_passed, content_dropped_empty_body`. Ordering rationale: Pre-Filter and Freshness are policy decisions, Content is a validity check; Content sits between them and the classifier LLM call — but **after** Pre-Filter only because Pre-Filter is title-only per ADR-0019 and does not need the body. _Avoid_: empty-body filter, body validator.

**Domain Pre-Filter**: A gate before the **Relevance Classifier**, dropping **Positions** whose **title** matches any **Negative Keyword** (title-only, blacklist-only, no whitelist — ADR-0019). Substring match, case-insensitive after `normalize()`. Owns its full operational protocol behind `admit(position) -> bool` / `emit_run_complete()`. Drops write status `out_of_domain` (same status the classifier writes for LLM-rejected items, per ADR-0020 renaming). Enumerates all matching keywords internally so it records per-position decisions to `pipeline_prefilter.transcripts.jsonl` and emits a per-keyword end-of-run summary to `pipeline_prefilter.events.jsonl`. Transcript record: `url, title, source, passes, reason ∈ {passed, blacklist_drop}, blacklist_matches, title_len`. Aggregate counts *positions matched* (not raw occurrences) and surfaces a `NEGATIVE_KEYWORDS_dead` list of zero-match keywords. Assumes input has been keyword-filtered upstream (parser invariant). _Avoid_: filter (ambiguous — see **Relevance Classifier**), gate, classifier.

**Quota Wall**: The shared coordination object the **Relevance Classifier**'s parallel worker pool checks before each call (ADR-0040). Implemented as a `threading.Condition` + `reset_time: datetime | None` guarded by the same lock. Surface: `raise_wall(reset_time)` (returns True iff this call was the first to raise the current wall — used to gate the single `quota_sleep` log row), `wait_if_blocked()` (blocks until past the wall; cheap no-op on the common path), `is_active()`. First worker to observe `ClaudeUsageLimitError` publishes `reset_time`; the orchestrator emits exactly one `event=quota_sleep` row per wall raise; remaining workers hit `wait_if_blocked()` at their next iteration and block on the same condition. In-flight calls at wall-raise time are allowed to finish — they will most likely 429 themselves, re-queue, and be sleep-blocked at next dequeue. With `claude_classify_parallelism = 1` the wall degenerates to a single-thread `sleep` (matches ADR-0023's original single-worker semantics). _Avoid_: rate limiter (no rate-shaping; only sleep coordination), barrier (not a one-shot sync point — the wall is repeatedly raisable across a run).

**Match Judge**: The call site that picks the **Daily Top-5** from today's **Pool**. Implemented as a single `LLMExtractor.judge_top_n(candidates)` per run — no wrapper class, no per-item call. Per ADR-0020 (amended by ADR-0041), takes `list[JudgeCandidate]` (stable id + **Header** + **Summary**) and returns up to 5 **Match Verdicts** (`{id, rank}`) — rank-only, no judge-produced summary. The judge ranks on Header + Summary prose; `raw_description` and (retired) `StructuredExtract` do not reach the judge prompt. **Skills** and the **Triage Profile**'s match-criteria reach the judge via `{skills}` and `{USER_INFO}` slots. On `ClaudeUsageLimitError` orchestrator sleeps and retries (ADR-0023). On any non-quota error the run fails without writing a daily file, and a **Failure Report** is emitted. Runs only on days the **Pool** is non-empty. Scale concern at large Pool sizes (~500 items) tracked in issue #524.

### Deduplication and run state

**Deduplication**: Skipping **Positions** seen in a previous run or earlier in the current run. Three-tier: an in-run ephemeral URL set (`RunScopedDedup`, obtained from the **Deduplication Store** via `run_scope()` context manager, sits in front of the persistent store to absorb the Cartesian overlap without paying duplicate `enrich()` cost) plus the persistent **Deduplication Store** with two tiers: exact-match on URL, plus exact-match on `(company_lc, title_lc, location_lc)`. The tuple tier fires only when **all three fields are non-`None`** on both sides. When the tuple matches under a new URL (syndicated copy), the store records an alias under the new URL — copying `status` and `first_seen` — so subsequent runs hit the cheap URL tier (ADR-0003). `first_seen` answers "when did this *role* first appear", not "when did this URL first appear". `is_seen` returns a `SeenResult`: `url_hit`/`tuple_hit` skip; `in_domain` enriches and routes directly into the **Pool** for today's judge call (no classify pass); `miss` processes from scratch. Alias write is performed inside `is_seen`.

**Dedup status enum** (per ADR-0020 / ADR-0022, supersedes the prior set):
- `out_of_domain` — written by **Domain Pre-Filter** (title hit) and **Relevance Classifier** (LLM `in_domain: false`). Terminal-skip.
- `in_domain` — written by classifier on `in_domain: true`, alongside the `{header, summary}` record in `extracts.json` (ADR-0041, supersedes the prior **Structured Extract** shape from ADR-0022). Means "in the **Pool**"; re-discovery routes directly into today's judge candidates.
- `selected_by_judge` — written after judge picks the item and the **Card** is appended+fsynced. Terminal-skip; extract deleted.
- `expired` — written by **Freshness Gate** when `posted_date` exceeds `MAX_LISTING_AGE_DAYS` or `deadline` < anchored date (ADR-0025, amended ADR-0041 — gate runs on both arms). Terminal-skip. Transitions from `in_domain` (also deletes the `{header, summary}` record from `extracts.json`) or any other status.
- `enrich_failed` — body fetch raised an unrecoverable HTTP/parsing error (e.g. per-URL 4xx wrapped by HTTP layer). Terminal-skip. _Note_: oversized stripped bodies and malformed LLM outputs do **not** write this status — they are re-checked next run per ADR-0041.
- ~~`external_redirect`~~ — _retired_ per ADR-0041 (which retired ADR-0013 + ADR-0037). Redirects are followed silently inside the **LLM Enricher**.

**Error semantics for `mark_*`** (orchestrator behavior, single-writer Pi):
- **Pre-filter drop** → `mark_out_of_domain(stub)` immediately. No LLM cost.
- **Classifier non-quota error**: batch fails; items NOT marked; orchestrator continues with other batches and still calls the judge on what classified successfully (ADR-0021). Next run retries.
- **Classifier `ClaudeUsageLimitError`** (ADR-0023): parse reset time, sleep until reset+2min (or fallback `next_top_of_hour + 2min`), retry the batch. No `degraded_reason`; no Failure Report. Cron-anchored day (ADR-0021) handles midnight crossings.
- **Judge non-quota error**: no daily file; **Failure Report** emitted; **Pool** intact (no `selected_by_judge` transitions); tomorrow sees today's pool plus tomorrow's arrivals.
- **Judge `ClaudeUsageLimitError`**: sleep and retry.
- **Per-card append succeeds, then judge processing fails mid-loop**: each card is appended+fsynced+`mark_selected_by_judge` atomically per winner. A failure between cards leaves earlier winners marked, later winners unmarked (back in pool for tomorrow) — bounded.
- **Body fetch raises unrecoverable HTTP error** → `mark_enrich_failed(stub)`. Terminal-skip. Includes per-URL 4xx (404/400/422 wrapped by HTTP layer; auth/5xx raise fatal and propagate, becoming `PARSER_DEAD`).
- **HTTP 3xx during body fetch** (ADR-0041, supersedes ADR-0037): the **LLM Enricher**'s HTTP client follows redirects silently up to its configured chain limit. No host classification, no `ExternalRedirect` payload. The destination's HTML is fed into the same strip path. If the destination has no usable body, **Content Gate** drops it (no `.seen.json` mark). If the destination is oversized, the raw HTML is stashed to `failures/oversized/` (no `.seen.json` mark).
- **Stripped body over token cap** (ADR-0041): no `.seen.json` write; raw HTML stashed to `<settings-dir>/failures/oversized/<source>-<url-slug>.html`; re-checked next run.
- **LLM output malformed** (empty `header`/`summary` when `in_domain: true`, missing tag, bad JSON; ADR-0041): no `.seen.json` write; raw output stashed to `<settings-dir>/failures/malformed/<source>-<url-slug>.txt`; re-checked next run.

State lives in `.seen.json` (synced via Syncthing alongside Daily Results Files, ADR-0002), shaped `{url: {company_lc, title_lc, location_lc, status, first_seen}}` (any of the lowercased fields may be `None`); fuzzy index built in-memory at load, not persisted. `DeduplicationStore` exposes six narrow methods (one per status: `mark_out_of_domain`, `mark_in_domain` — takes the extract as kwarg, `mark_selected_by_judge`, `mark_expired`, `mark_enrich_failed`, `mark_external_redirect`). Pipeline is single-writer at the process level (ADR-0002); the lock inside `DeduplicationStore` (ADR-0014) covers concurrent classify/judge worker writes. Parser threads remain pure producers. **Migration**: ADR-0024 wipes the previous `.seen.json` and the old trio; no automatic translation. _Avoid_: duplicate filtering, URL filtering.

### Display

**Card**: The rendered markdown block per **Daily Top-5** winner, produced by the **Renderer**. Fixed two-block structure per ADR-0042: `# **{rank}:** {Header}` followed by a blank line and `{Summary}` (each terminated by `\n`). `{rank}` is the Judge-assigned 1..5 position; `{Header}` and `{Summary}` are the LLM-authored strings persisted in `extracts.json` (ADR-0041). No placeholder vocabulary beyond rank; no user-tunable layout (ADR-0042 retires `Layout`). _Avoid_: card template, headline.

**Renderer**: A one-line concatenation: `render(rank: int, header: str, summary: str) -> str` returns `f"# **{rank}:** {header}\n\n{summary}\n"`. No `Position` argument, no `MatchVerdict` argument, no `Layout` argument. Per ADR-0042 (supersedes ADR-0004). _Avoid_: formatter, presenter.

**Results File Manager**: The only module that reads or writes the **Daily Results File**. Function-shaped: orchestrator holds the day's `Path` (`<settings-dir>/results/{cron_anchored_date}.md`, ADR-0021) and dispatches. Surface: `ensure_initialized(path)` (`mkdir(parents=True)`; no header — files start empty), `append(path, rendered_block)` (verbatim write + `flush` + `os.fsync`, propagates `OSError`). `data_dir` is the parent of the loaded `config.py` (ADR-0011). _Avoid_: writer, output manager.

**Atomic Write Helper**: A single-function module providing crash-safe atomic overwrite of a file from a `bytes` payload. Interface: `write_atomic(path: Path, payload: bytes) -> None`; raises `OSError` (unwrapped) on any failure. Implementation invariant: opens a `.tmp`-suffix sibling of `path` (named by the helper — callers do not supply a tmp path), performs `os.write` → `os.fsync` → `os.close` → `os.replace` with mode bits `0o644`. A `.tmp` file on disk after a run is always a crash signal — successful writes leave no `.tmp` artefact behind. Encoding is the caller's responsibility; callers pass `payload.encode("utf-8")` (the helper is format-agnostic). No locking: thread-safety stays with the caller; ADR-0002 (single-writer) and ADR-0014 (per-store lock) remain the sole concurrency mechanisms. Used by the **Deduplication Store** and the **Extract Store** from their `_persist` methods. The **Failure Report** writer's weaker `tmp+replace` variant (no `fsync`) is a known future adoption target. _Avoid_: persistence helper (overloaded), file writer (overloaded with the **Results File Manager**'s append idiom — which is not atomic-replace and does not use this helper).

### Observability

**Log Artifacts**: Files under `logs/`, laid out **by reader** (ADR-0018). Four types: `<comp>.events.jsonl` (one structured row per step; the run-end metrics row that used to live on the retired Run Divider now lands as `event=run_complete` here per ADR-0021); `lifecycle.jsonl` (single shared file carrying status-display `registered`/`phase_changed`/`removed` events); `run.log` (single shared file carrying tracebacks and `SUMMARY OF SESSION` blocks); `<comp>.transcripts.jsonl` (LLM components only — full prompt/response payloads). Every component identifier carries a layer prefix — `parser_`, `llm_`, `pipeline_`. For LLM code, **component is the LLM-stage row**, not the implementation class — `llm_classify_relevance` for the per-item classifier and `llm_judge_match` for the end-of-run top-N judge (the identifier predates ADR-0020's switch from per-item `judge_match` to single-call `judge_top_n` and is kept for log-file continuity). _Avoid_: log file (without qualifier), debug log.

**Run Log**: The per-run instance that writes the **Log Artifacts** to `logs/`. Constructed once at orchestrator entry from `cfg.logs_dir`; threaded as a kwarg into every component that emits log events (`RunMetrics`, `ClaudeExtractor`, the status-display core, the outbound dispatcher, every **Parser** via context-manager construction which forwards into its `ParserHttp`). Owns the directory path, file-naming convention, JSONL envelope shape, and `=== <component>  <ts>  <kind> ===` header convention in `run.log`. Filesystem errors propagate as `OSError`. Safe to share across threads — each method opens, writes, closes per call. _Avoid_: log writer, logger (overloaded with `logging.Logger`).

**Status Display**: A live, in-process view of pipeline progress. Ephemeral — exists only while `run()` executes. A `StatusDisplay` Protocol with two implementations: `RichStatusDisplay` (`rich.Live` table repainting at ~4 Hz, when `sys.stdout.isatty()`) and `PlainStatusDisplay` (line-by-line `print`, Pi cron). One row per stage: overall `pipeline` phase, transient `startup`, one **Agent Row** per configured **Parser** (`parser_<parser_type>`), and rows for `pipeline_dedup`, `pipeline_prefilter`, `pipeline_freshness`, `llm_classify_relevance`, `llm_judge_match`. Body strings formatted by **RunMetrics**. `register()`/`update_phase()` also emit to `lifecycle.jsonl` (ADR-0018). _Avoid_: progress bar (implies known total), TUI, dashboard.

### Sources & extraction

**Source**: A configured job board or API in `Config.SOURCES`, declared as a `SourceEntry` carrying a **Parser Type** and per-source `max_results`. A **Source** is a *config entry* (what to search); a **Parser** is the *module* (how). One-to-one in v1. _Avoid_: site, board.

**Parser**: A Python module in `src/application_pipeline/parsers/`. Single-phase API per ADR-0041: `discover(query) -> Iterable[PositionStub]` plus an optional `body_selector: str | None` (single CSS selector identifying the source's job-body container, consumed by the **LLM Enricher**). The previous two-phase `enrich()` method retires — body fetch and field extraction move to the shared **LLM Enricher**. Each parser is a context manager owning a single `httpx.Client`; no shared mutable module-level state (ADR-0005). The orchestrator owns Cartesian expansion of `KEYWORDS × LOCATIONS [+ remote]` and emits one `discover(ParserQuery(keyword, location, max_results))` per combination; `location` is sealed **Location** (`City(name) | Remote`). Each parser declares its **Location Coverage** as module-level symbols (ADR-0012). Parsers own discover-phase per-host pacing; the **LLM Enricher** owns body-fetch pacing for its generic HTTP client. **Keyword-match invariant** (ADR-0019): every stub yielded from `discover()` is expected to match its `ParserQuery.keyword` in the title. _Avoid_: scraper, fetcher.

**Position Stub**: The result of `Parser.discover()` — `url, title, source, posted_date?, company?, location?`. Required: `url, title, source`. Optional (`| None`): `posted_date: date` (drives the pre-LLM arm of the **Freshness Gate** per ADR-0041), `company: str`, `location: str` (Header pre-fills passed into the **LLM Enricher**; LLM substitutes them verbatim or supplements from the body). _Avoid_: preview, summary.

**External Redirect**: _Retired_ per ADR-0041 (which supersedes ADR-0013 and ADR-0037). Redirects are followed silently inside the **LLM Enricher**'s body fetch. _Avoid_: do not reintroduce as a payload type; the `external_redirect` `SeenStatus` and `external_redirects` counter are also gone.

**Parser Type**: The string on a `SourceEntry` identifying which **Parser** to use — maps directly to a filename in `parsers/`. _Avoid_: adapter.

**Location**: Geographic slot the orchestrator passes via `ParserQuery.location`. Sealed sum type: `City(name: str)` for a named place from `Config.LOCATIONS`, `Remote` for the remote-only slot when `INCLUDE_REMOTE=True`. _Avoid_: place, geo.

**City**: The `City(name)` arm. `name` is the user-typed string, normalized via `normalize()` before lookup. No central catalog — resolvable city names are the dynamic union of configured **Sources**' `serves()` predicates. _Avoid_: town, place.

**Remote**: The `Remote` arm. Each **Parser** decides what `Remote` means for its **Source** via `remote_wire()`. _Avoid_: homeoffice, work-from-home (when the type variant is meant).

**Location Coverage**: The module-level **Protocol** every **Parser** satisfies — `serves(name: str) -> bool`, `to_wire(name: str) -> str`, `serves_remote: bool`, `remote_wire() -> Any`. Each parser is local authority. **Config Loader** validates `LOCATIONS` at load time and rejects unresolvable entries with `ConfigError`. Per ADR-0012. _Avoid_: location map, slug table.

**LLM Extractor**: A Protocol with two methods — `classify_relevance(item: ClassifyItem) -> (RelevanceVerdict, CallUsage)` and `judge_top_n(candidates: list[JudgeCandidate]) -> (list[MatchVerdict], CallUsage)`. Per ADR-0041: `ClassifyItem` carries `title`, `raw_description` (stripped body), and any parser-pre-filled `PositionStub` Header fields. `RelevanceVerdict` is `{in_domain: bool, header: str | None, summary: str | None}` (header/summary non-None iff in-domain). `JudgeCandidate` carries stable id, **Header**, **Summary**, and the `PositionStub` fields for tie-breaking. `MatchVerdict` is `{id, rank}` — rank-only (ADR-0041); the prior `matched`/`missing`/`summary` fields retire. Hardcoded prompts live in `src/application_pipeline/templates/prompts/`; **Triage Profile** injected via `{USER_INFO}` slot at load time; **Skills** bound at construction (judge only). The v1 production implementation is `ClaudeExtractor` shelling out to **Claude Code CLI** (`claude -p - --output-format json --model <alias>` + `--effort <level>` where applicable, plus `--no-session-persistence --disable-slash-commands --tools "" --setting-sources user` per ADR-0038) as a fresh subprocess per call. The full combined prompt is sent as a single message via stdin (ADR-0038); `--system-prompt` is not used. Per ADR-0015: `haiku` for the classifier, `haiku` with `--effort medium` for the judge (model choice may be revisited under the more-capable-LLM regime). Structured outputs wrapped in `<verdict>` / `<verdicts>` tags and parsed via **Agent Output Protocol** (ADR-0015). On `ClaudeUsageLimitError` the implementation parses the 429 reset time and raises `ClaudeUsageLimitError(reset_time)` so the orchestrator can sleep (ADR-0023). _Avoid_: LLM, model (without qualifier).

**Agent Output Protocol**: A project-agnostic pure module under `application_pipeline/llm/` that extracts a structured JSON payload from a CLI response whose `result` carries free-form text with payload wrapped in a semantic XML tag (ADR-0015). Exposes `extract_json_block(text: str, tag: str) -> Any` + `AgentOutputProtocolError(kind: "tag_missing" | "json_malformed")`. Finds rightmost `</tag>`, walks back through preceding `<tag>` openings, strips an optional surrounding markdown fence, then `json.loads`. Returns the first candidate that parses; raises on no tag / no parse. The **LLM Extractor** catches at its call-site boundary and re-raises as `ExtractorBatchMalformedError`/`ExtractorMalformedJSONError`. _Avoid_: output parser (too generic), response handler (overloaded with HTTP).

**Pagination**: Successive page fetches per **Source**, preferably ordered newest-first by `posted_date`, until either the source signals no next page **or** the per-Source `max_results` cap is reached. Per ADR-0017 there is no dedup-driven discover early-stop. `SKIP_AND_END_QUERY` (ADR-0007) is only emitted on `run_state.is_aborted`. Each `SourceEntry` carries its own cap (default 1000).

### Invocation

The package is distributed via PyPI (ADR-0027). Hosts install into a project-local `.venv` (`python3 -m venv .venv && .venv/bin/pip install application-pipeline`), bootstrap the settings folder with `application-pipeline init`, and arm cron with `bash application-pipeline/setup/cron-install.sh`. **Home dir is hardcoded to `<cwd>/application-pipeline/`** — no env-var override, no auto-discovery (ADR-0029, mirrors pycastle's CWD-relative pattern). The three subcommands take no settings-dir argument:

- `application-pipeline run` reads `<cwd>/application-pipeline/config.py`.
- `application-pipeline init [--refresh]` seeds into `<cwd>/application-pipeline/`.
- `application-pipeline compile-cv <app_dir>` compiles a per-listing draft. Reads `<app_dir>/cv.tex` (slot-map written by `/write-cv` — see **CV Slot-Map** and **CV Skeleton**), substitutes its slots into the package's `cv_template.tex`, `\input`s `cv/facts.tex` from `<cwd>/application-pipeline/user-info/`, writes the result to `<app_dir>/.build/` and runs `pdflatex` for the three `\BUILD` variants. Windows paths are normalized to POSIX (`as_posix()`) before injection into `\def\CvDataDir{...}` (renamed from `\UserDataDir` per ADR-0032; now points at `user-info/cv/`) to avoid TeX swallowing backslashes as control sequences (ADR-0030).

`run` and `compile-cv` fail loud-and-fast (exit 2, named error) at the CLI boundary if `<cwd>/application-pipeline/config.py` is missing — the message names both fixes ("did you forget to cd, or run init?"). Cron fires **weekdays at 00:30 local** (ADR-0024, amended). Each tick runs the seeded `setup/cron.sh`, which `cd`s to the project root via `$(dirname "$0")/../..` (self-locating; cron line carries only the absolute path to `cron.sh`) before invoking: `pip install --upgrade application-pipeline` (×2 for CDN propagation), `application-pipeline init --refresh`, then `application-pipeline run`. **`init` semantics** (refined post-ADR-0011, amended ADR-0042): plain `init` seeds-if-missing for everything; `init --refresh` overwrites `setup/*.sh` unconditionally (the only refreshable subtree), **deletes** `layout.py` if present (ADR-0042 retires layout), and seeds-if-missing for everything else (`config.py`, `user-info/*` — never overwritten). User-authored files are safe across upgrades; new global template files added in a release self-heal onto the host. A global `flock` at `<cwd>/application-pipeline/.cron.lock` (modelled on pycastle's pattern) serialises overlapping ticks; a run that sleeps through a quota window (ADR-0023) may overlap the next cron fire, which the flock silently waits out — absence of the next weekday's daily file is the operator signal. The laptop is used only during development to smoke-test individual **Parsers** in isolation. Note: `pycastle/` in this repo is an unrelated RALPH Loop coding-agent plugin used to *build* this project; its quota-handling logic (`pycastle/services/claude_service.py:88-145`) is the reference port for ADR-0023's parse-and-sleep behavior, and its `setup/*.sh` scripts are the reference shape for ADR-0027's seeded cron wrappers.

## Relationships

- A listing is written into a **Daily Results File** only if it passes (in order): the **Domain Pre-Filter** (title-only), the pre-LLM arm of the **Freshness Gate** (when the parser pre-filled `posted_date`), the **LLM Enricher**'s body fetch + strip + token-cap check, the **Content Gate** (empty stripped body), the post-LLM arm of the **Freshness Gate** (when LLM inferred `posted_date`), the **Relevance Classifier** (status → `in_domain`, `{header, summary}` persisted to `extracts.json`), then is picked into the **Daily Top-5** by the **Match Judge** (status → `selected_by_judge`). The post-LLM arm of the **Freshness Gate** and the **Content Gate** both re-run on Pool re-discovery, so pool items can transition `in_domain → expired` (deletes the extract record) or be re-checked when the body is missing.
- The **Pool** is the set of `status == in_domain` URLs re-discovered by a parser in the current run — an emergent property of `.seen.json` + today's parse output, not a stored data structure.
- The **Match Judge** runs once per run on the **Pool**, takes **Header** + **Summary** strings per candidate plus construction-bound **Skills** and the injected **Triage Profile**, returns up to 5 **Match Verdicts** carrying only `{id, rank}` (ADR-0041).
- The **Triage Profile** reaches the LLM via the hardcoded prompt's `{USER_INFO}` slot (ADR-0016); **Skills** are bound at `ClaudeExtractor` construction and reach the LLM via `{skills}` (judge only); `NEGATIVE_KEYWORDS` reaches the **Domain Pre-Filter** directly (ADR-0019).
