# Application Pipeline

Personal job-discovery and triage pipeline. Fetches listings from a small set of sources, classifies relevance with Claude, accumulates an in-domain **Pool** across days, emits one dated **Daily Results File** per day carrying the **Daily Top-5** **Cards** ranked by the **Match Judge**. Application authoring (CV/cover letter) is out of scope for v1.

## Scope

- **In scope (v1):** Working **Parsers** for **Bundesagentur**, **stellen.hamburg**, **jobs-beim-staat**, smoke-tested standalone on the laptop.
- **In scope (v1.1):** Full pipeline on Pi — orchestrator, **LLM Enricher** (per ADR-0041 — body fetch + strip + **Relevance Classifier** producing **Header** + **Summary**), **Match Judge** (one call per run, picks **Daily Top-5** from the **Pool**), **Deduplication**, daily file, cron once per day, Syncthing sync.
- **Out of scope:** **Profile** ingestion, additional commercial parsers, browser automation. (CV / cover-letter LaTeX compilation now in scope via `application-pipeline compile-cv` — see Invocation.)

## Language

### Pipeline artifacts

**Config**: A `config.py` at the root of the settings directory, hardcoded to `<cwd>/application-pipeline/` (ADR-0029) controlling pipeline shape — `SOURCES`, `LOCATIONS`, `INCLUDE_REMOTE`, Claude settings (optional `claude_cli_path`), `MAX_LISTING_AGE_DAYS: int` (default 180, `≥ 1`) driving the **Freshness Gate**'s `posted_date` arm, and `claude_classify_parallelism: int` (default 4, `≥ 1`) sizing the **Relevance Classifier**'s parallel classify-only worker pool (ADR-0040, amended ADR-0051 — workers now drain the classify queue and run only the LLM call, no body fetch). The former `claude_classify_batch_size` knob is retired (ADR-0036 supersedes ADR-0014's batching). No path override knobs: `user-info/` and prompt locations are all canonical under the settings dir (ADR-0011 amended). `layout.py` retired per ADR-0042. **Does NOT** carry `KEYWORDS`/`SKILLS`/`NEGATIVE_KEYWORDS` — those live in **SearchTerms** (ADR-0028). Plain Python literals; `#` comments. Materialised on first bootstrap from `src/application_pipeline/templates/config.py` via `application-pipeline init <dir>` (ADR-0011); the user edits in place from then on. Never overwritten by `init --refresh` (ADR-0011, refined: refresh overwrites `setup/` only). Loaded by **Config Loader**, returns a frozen typed `Config`. ~~`max_results`~~ is _retired_ per ADR-0050 — parsers paginate until the source returns an empty page. Cron schedule lives in crontab, not Config (ADR-0024 amended, weekdays at 00:30 local). _Avoid_: config file, settings file, search config.

**SearchTerms**: The user-authored search/filter knobs lifted out of **Config** per ADR-0028 — `KEYWORDS`, `SKILLS`, `NEGATIVE_KEYWORDS`. Lives as three markdown files under `<settings-dir>/user-info/search-terms/` — `keywords.md`, `skills.md`, `negative-keywords.md` (ADR-0032; supersedes the prior single-file layout). Filename *is* the section — no `##` header inside each file. Body shape: flat `-` bullet entries; `keywords.md` and `negative-keywords.md` may carry topical sub-labels as plain non-bullet lines, and `skills.md` additionally carries H2 **Skill Group** headings with pandoc-style `{...}` attributes (ADR-0033). Loaded by a dedicated loader (one flat-bullet parse per file — headings and attribute blocks in `skills.md` are ignored at the pipeline parse layer), exposed as a frozen `SearchTerms` object threaded alongside `Config` into the orchestrator, **Domain Pre-Filter**, and `ClaudeExtractor` construction. `keywords.md` missing OR present-but-zero-bullets raises `SearchTermsError` (degenerate empty pipeline); `skills.md` and `negative-keywords.md` are optional — missing file treated as empty list. No length validation on negative keywords (lifted in ADR-0028; was `≥ 3` when on Config). Distinct from the **Triage Profile** — they share the `user-info/` tree but live in different sub-dirs and have different downstream consumers (SearchTerms drives parser orchestration via KEYWORDS, the Domain Pre-Filter via NEGATIVE_KEYWORDS, and an LLM prompt's `{SKILLS}` slot via SKILLS — currently the judge only, though the slot mechanism is generic and the classifier could plug into it; Triage Profile is LLM-prompt content via `{SELF_DESCRIPTION}` / `{MATCH_CRITERIA}`). Hard cutover: installs with the pre-ADR-0032 layout (single `search-terms.md` at user-info root) error on first run until the three files exist under `search-terms/`. _Avoid_: search config, term file.

**Layout**: _Retired_ per ADR-0042. The `layout.py` file, `Layout` dataclass, placeholder vocabulary, `PLACEHOLDER_GROUPS` / `CARD_TEMPLATE` constants, named-group separator logic, and `str.format_map` plumbing are all gone. Card structure is hardcoded in the **Renderer** (one-line concatenation prepending `# **{rank}:** ` to the LLM-authored Header + Summary). `init` no longer seeds `layout.py`; `init --refresh` deletes it from existing installs. `LayoutError` retired. _Avoid_: do not reintroduce as a knob — Header content is shaped by the classify prompt, not a runtime template.

**Daily Results File**: One dated markdown file per calendar day at `<settings-dir>/results/YYYY-MM-DD.md`, holding the **Daily Top-5** as **Cards** in **Rank** order. Date is **cron-anchored** (ADR-0021/0023) — a run started day X writes to day X's file even if it crosses midnight sleeping through quota. No `FILE_HEADER`, no preamble, no **Run Divider**. If today's pool had fewer than 5, the file carries however many cards; if pool was empty, no file is written. Write-once on the host; if a sync channel is configured (e.g. Syncthing), the file propagates outward read-only. _Avoid_: results file (without "daily"), `current.md` (gone), dated results file.

**Failure Report**: A markdown file at `<settings-dir>/.runtime-data/failures/<timestamp>.md` written on a failed run or a parser-level error — `cron.sh` pipeline-stage errors (ADR-0027; `init --refresh` or `run` exits non-zero — pip-upgrade failures warn-and-continue instead), orchestrator runtime errors, **Match Judge** failure (no daily file), non-quota classifier errors that left the run with no writeable output, **parser-dead events** (ADR-0050 — includes parser ID, URL, HTTP status or exception, traceback), **missing title in discover results** (ADR-0050 — API schema may have changed), and **native enrich failures** (ADR-0049 — empty body, auth failure, 3xx redirect, JSON decode error on the detail API). Per ADR-0010 (path updated by ADR-0046). Each Failure Report must include enough context for the operator to diagnose the problem without reading `run.log`; different failure types must be distinguishable at a glance. Acknowledged by deleting the file. Quota errors do NOT trigger a Failure Report — they sleep until reset (ADR-0023). Per-listing soft failures (oversized stripped body, malformed LLM output; ADR-0041) do **not** trigger Failure Reports either — they stash to sibling directories `<settings-dir>/.runtime-data/failures/oversized/` and `<settings-dir>/.runtime-data/failures/malformed/` with no `seen.json` write, and the URL is retried next run. _Avoid_: incident report, error log.

**Position**: A single job listing surviving the **Relevance Classifier** and the **Match Judge**'s top-5 selection. Identified within its **Daily Results File** by its rendered **Card**. The Card shape is fixed (ADR-0042): `# **{rank}:** {Header}` followed by `{Summary}`. _Avoid_: job, listing, vacancy.

**Position Schema**: _Retired_ per ADR-0041. Parsers no longer construct a `Position`. The closed-enum fields (`salary`, `contract_type`, `employment_type`, `work_model`, `deadline`) are no longer extracted as separate fields — their content lives inside the LLM-authored **Header** string. `PositionStub` and `EnrichResult` survive in `parsers/types.py` (the latter added by ADR-0047 when parser-side `enrich()` was restored as a body-producer — *not* a resurrection of per-field extraction, which stays retired). The "parsers never guess" invariant retires with this dataclass; the **LLM Enricher** is allowed to infer from body text (capable-model regime, 2026-05-22 grilling). _Avoid_: do not reintroduce per-field extraction in parsers.

**Raw Description**: The full body text of a candidate listing, fetched and stripped by the **LLM Enricher** (per-source CSS selector when available via `Parser.body_selector`, generic library fallback — `trafilatura` / `readability-lxml` — for redirect destinations and selectorless sources). Per ADR-0041, raw_description is **fed to the LLM as classify input but never persisted and never rendered into a Card**. Empty stripped body is dropped by the **Content Gate** (ADR-0039) before the LLM call; the URL is not marked in `seen.json` so the next run re-fetches. Over-cap stripped bodies are dropped, the raw HTML is stashed to `<settings-dir>/.runtime-data/failures/oversized/<source>-<url-slug>.html`, and `seen.json` is not marked (re-check next run, same pattern as Content Gate). _Avoid_: description (when full text is meant); not to be confused with the LLM-authored **Header** / **Summary**.

**Structured Extract**: _Retired_ per ADR-0041 (which supersedes ADR-0022). Replaced by **Header** + **Summary** — both LLM-authored strings produced by the classifier alongside `matches: true` (per ADR-0043). `extracts.json` is `{stable_id: {header, summary}}`. Wipe-state migration per ADR-0024 — no auto-translation from the prior shape. _Avoid_: do not reintroduce; field-level extraction is gone.

**Header**: The three-line block authored by the **LLM Enricher** at classify time (ADR-0041) — title on line 1, `company · location · work_model` on line 2, `posted_date · seniority · salary` on line 3. The LLM substitutes a known value when the parser pre-filled a `PositionStub` field, infers from the body when it can, or drops the segment (and its `·` separator) when neither parser nor body offers a signal. Persisted as a single string in `extracts.json`. Rendered into the **Card** verbatim under the rank prefix. _Avoid_: card top, headline.

**Summary**: The prose paragraph authored by the **LLM Enricher** at classify time (ADR-0041), describing the role in the **Triage Profile**'s frame. Persisted alongside **Header** in `extracts.json`. Rendered into the **Card** below the Header. The **Match Judge** ranks on Header + Summary directly — no structured comparison fields. _Avoid_: match verdict summary (retired with `MatchVerdict.summary` per ADR-0041), description (overloaded with **Raw Description**).

### Filtering & scoring

**Triage Profile**: The applicant's self-description plus rules deciding whether a role matches. Lives in `<settings-dir>/user-info/triage-profile/` (ADR-0032, amended ADR-0043) as three markdown files — `self-description.md` (who the candidate is), `match-criteria.md` (what makes a role a fit — covering both domain scope *and* preferences after ADR-0043 merged in the retired `domain-fit.md`), and `writing-style.md` (PRD-#16-v2.1 authoring skills only, NOT injected into v1 prompts). Per ADR-0043 (supersedes ADR-0016's per-call-site routing; amended post-#535), the loader exposes three named profile slots — `{SELF_DESCRIPTION}` (body of `self-description.md`), `{MATCH_CRITERIA}` (body of `match-criteria.md`), `{SKILLS}` (skills bullets from **SearchTerms**) — that each prompt template places with its own heading. Classifier and judge both consume `{SELF_DESCRIPTION}` + `{MATCH_CRITERIA}`; only the judge consumes `{SKILLS}` (ADR-0019). The classifier uses flat H1; the judge keeps its H2 sub-headers. Bullets/keywords, German, "extremely concise, sacrifice grammar for concision" (ADR-0026). Reused as v2 authoring context — no separate CV Profile. _Avoid_: profile (without qualifier), bio, CV Profile (retired before being built — do not reintroduce), `domain-fit.md` (retired, content merged into `match-criteria.md`).

### CV authoring

**CV Slot-Map**: The per-listing file `<app_dir>/cv.tex` emitted by the `/write-cv` skill and consumed by `compile-cv` (ADR-0030). Sectioned-marker syntax: `^%% SLOT: <name>$` introduces a body running until the next `%% SLOT:` line or EOF. Bodies are raw TeX fragments (multi-line German prose with `\href`, `\textit`, umlauts — no escape rules). Thirteen committed slots: `recipient_company/name/street/zip_city` (DIN 5008 Anschriftenfeld, four lines — per issue #472), `opening`, `cover_intro/pivot/fit/closing`, `resume_berufserfahrung/ausbildung/projekte` (per issue #453 — supersedes single `<<RESUME_BODY>>`), `skills_block` (mechanically assembled by `/write-cv` from the **Skill Group** pool in `skills.md`, not freehand-authored — ADR-0033; the slot body carries the rendered `\cvitem` rows verbatim so `cv.tex` remains the self-contained artifact). Everything else (name/address/photo/languages/hobbies/title) is listing-invariant and lives in **Facts**. _Avoid_: cv document (CV Slot-Map is the *input*, not the document; `cv_template.tex` is the document).

**CV Skeleton**: The format-by-example file at `<settings-dir>/cv-template/cv_skeleton.tex` (ADR-0030, relocated by ADR-0044). Package-shipped via `src/application_pipeline/templates/application-pipeline/cv-template/cv_skeleton.tex`, seeded by `init`, **refreshable** by `init --refresh` (ADR-0011 amendment). Carries the slot list + per-slot prompt guidance. The `/write-cv` skill reads the skeleton from the settings dir (never from `src/`) and injects it into the prompt as the output-format example. The skill's interface contract is the settings dir; the skill stays agnostic of install location. `init --refresh` deletes the obsolete `<settings-dir>/skills/` directory if present (the prior host location). _Avoid_: cv template (overloaded with `cv_template.tex` — the skeleton is a *prompt example*, not the LaTeX document).

**Facts**: The listing-invariant data file at `<settings-dir>/user-info/cv/facts.tex` (ADR-0030; relocated under `cv/` per ADR-0032; restructured per ADR-0035). Defines `\myFirstname`, `\myFamilyname`, `\myCity` (raw scalars used by the template across cover + resume), plus three presentation macros — `\PersonalInfo`, `\Languages`, `\Hobbies` — each a block of `\cvitem` / `\cvlistitem` calls the user freely edits to add or drop rows. `cv_template.tex` `\input`s facts via `\CvDataDir/facts` and calls `\firstname{\myFirstname}`, `\familyname{\myFamilyname}`, `\title{Lebenslauf}`, `\photo[120pt][1pt]{\CvDataDir/profile}`; the resume's Persönliche Informationen section emits `\cvitem{Name}{...}` then `\PersonalInfo`. Replaces the previous `identity.tex` + `contact.tex` split. _Avoid_: identity (subset only), contact (subset only) — both retired as filenames; the per-field raw defs (`\myStreet`, `\myZip`, `\myPhone`, `\myEmail`, `\myGithub`, `\myLinkedin`) and the `\Xdisplay` mirror defs are also retired (folded into `\PersonalInfo`).

**Content Pool**: The pool of CV item macros at `<settings-dir>/user-info/cv/content_pool.tex` (relocated under `cv/` per ADR-0032; `cv_template.tex` `\input`s it via `\CvDataDir/content_pool`). Each `\newcommand{\itemFoo}{...}` is a `\cventry`/`\subcventry` body the `/write-cv` skill can select into a `resume_*` slot. Per-item metadata header (ADR-0030) is three fields: `always:` (boolean — must appear in every application), `group:` (string — mutually exclusive variants), `relevance:` (`<jobtype>=<high|medium|low>, ...`). Sections (`Berufserfahrung` / `Ausbildung` / `Projekte`) are derived from `% ===== <name> =====` block headers above each item, not stored per-item. The previous `section:`, `tags:`, `summary:` fields are retired (section was duplicate of position; tags overlapped with relevance; summary drifted from `\cventry` body — Claude reads bodies directly). _Avoid_: item pool, CV pool, content database.

**Skill**: A hard-skill or technology item from **SearchTerms**' `SKILLS` (ADR-0028; previously on Config). Dual-consumed (ADR-0033): the pipeline harvests every bullet body in `skills.md` as a flat list, baked into the **Match Judge** prompt's `{SKILLS}` slot at `load_prompts` time (per ADR-0043 amendment); `/write-cv` reads the same file with its full **Skill Group** + attribute structure to assemble the CV's `skills_block` slot. Per ADR-0019, `SKILLS` is **only** consumed by the judge prompt — the **Domain Pre-Filter** no longer reads it. The LLM's `matched`/`missing` lists are open-vocabulary. An item carries an optional `{always}` attribute (within-group render floor; does **not** promote its group into the CV — ADR-0033). _Avoid_: keyword (when matching).

**Skill Group**: An H2 heading in `skills.md` (ADR-0033) — also the `\cvitem{<group>}{...}` category label in the rendered CV. The unit of LLM selection for `/write-cv`: always-groups (`{always}`) always render; non-always groups are picked by the LLM against the listing's jobtype via per-group `<jobtype>=<high|medium|low>` relevance entries. Within an included group every `{always}` **Skill** renders plus zero-or-more LLM-picked items; excluded groups contribute nothing. File order = render order; the LLM does not reorder groups or items. Empty included groups (zero `always`, zero picks) collapse silently — no empty `\cvitem` row. _Avoid_: skill category (the prior judge `{matched}`/`{missing}` open-vocab lists are retired per ADR-0041, so no longer a clash — but the term still overloads), section (overloaded with **Content Pool**'s `% ===== <name> =====` block headers).

**Agent Skills**: The Claude-Code agent workflows shipped with the package and seeded onto the host under `<cwd>/.claude/skills/` (ADR-0044). Four directories: `_shared/` (cross-skill protocol — startup checks, triage routing, slot-map conventions), `analyse-listing/` (per-listing grilling that writes `analysis.md`), `iterate-cv/` (conversational CV revisions, hot-loop), `write-cv/` (generate `cv.tex` + cover/resume/combined PDFs). Source of truth lives in `src/application_pipeline/templates/claude/skills/`; the repo root no longer carries `.claude/skills/` directly — it is resurrected by `init --refresh`. Refresh overwrites every file inside the four package-owned skill dirs; user-added skill dirs (e.g. `<cwd>/.claude/skills/my-skill/`) and operator-added files inside package-owned dirs are not deleted. Skills read pipeline state from `<cwd>/application-pipeline/user-info/...` and assume Claude is launched from `<cwd>` (or a subdirectory). _Avoid_: subagents, prompts (overloaded with the LLM-extractor's `templates/prompts/`).

**Keyword**: A search term used to query a **Source**, from `SearchTerms.KEYWORDS` (ADR-0028; previously `Config.KEYWORDS`). Distinct from a **Skill** (does not affect judgment) and from `NEGATIVE_KEYWORDS` (which drives the **Domain Pre-Filter**). _Avoid_: skill (when querying), negative keyword (when querying).

**Negative Keyword**: An entry in `SearchTerms.NEGATIVE_KEYWORDS` (ADR-0028; previously on Config). Patterns that, if found via case-insensitive substring match in a **Position**'s **title** (body text not consulted, per ADR-0019), cause the **Domain Pre-Filter** to drop the listing. No length validation (lifted in ADR-0028). No rescue mechanism. _Avoid_: blacklist, exclusion.

**Match Verdict**: The structured output of the **Match Judge** for each of the 5 winners. Per ADR-0041, simplified to `{id, rank: 1..5}` — the Judge ranks only. The Card's Summary text is the classify-time **Summary**, not a Judge-produced summary; `matched` / `missing` lists retire (rank-signal is implicit in the Header/Summary the Judge reads). _Avoid_: score, rating, tier (retired).

**Rank**: The `rank` field — integer 1..5 assigned by the judge when picking the **Daily Top-5**. Surfaced in the **Card** via `{rank}` placeholder. Not a score (no cross-day comparability); not a tier (no enum). _Avoid_: tier (retired), score, position (overloaded).

**Pool**: The implicit set of matched **Positions** eligible for today's **Match Judge** call — every URL in `seen.json` with `status == matched` re-discovered by a parser in the current run. No separate data structure; computed per-run as `{url ∈ discovered_today : seen.json[url].status == matched}` — re-discovery alone is enough; the URL-tier dedup short-circuits enrichment, so Pool re-entry costs nothing beyond the discover hit and the judge sees the persisted Header/Summary from the original classify run. Enter: classified `matched` (status flips, extract written). Exit: judge picks it (status → `selected_by_judge`, extract deleted) or no longer re-discovered (record sticks but never re-enters). No pool size cap, no TTL eviction in v1 (ADR-0022). _Avoid_: queue, candidate set, judge list — and not to be confused with **Daily Top-5** (the pool is the input set; the top-5 is the selected output).

**Daily Top-5**: The 5 **Positions** the **Match Judge** returns from its single end-of-run call, drawn from today's **Pool**. ≤5 if pool was smaller. Surfaced as **Cards** in **Rank** order. Each winner gets `mark_selected_by_judge(stub)` after its **Card** is appended+fsynced, removing it from the **Pool**. _Avoid_: top-N, shortlist.

**Relevance Classifier**: The classify-time LLM call site, subsumed into the shared **LLM Enricher** per ADR-0041 and tightened into a three-check gatekeeper per ADR-0043. Decides whether a candidate listing passes all three of: (1) in the candidate's domain, (2) within the candidate's realistic skill/experience floor, (3) fits the candidate's preferences (role type, seniority, hard no-gos). When all three pass, emits the rendered **Header** and **Summary** for the Card. Implemented as `LLMExtractor.classify_relevance(item: ClassifyItem) -> (RelevanceVerdict, CallUsage)`; output shape `<verdict>{"matches": true, "header": "...", "summary": "..."}</verdict>` or `<verdict>{"matches": false}</verdict>` (ADR-0043). One `claude -p` subprocess per listing (per ADR-0036, amended by ADR-0041 — only the payload shape changes). Per ADR-0051, the parallel classify worker pool (N = `Config.claude_classify_parallelism`, default 4 per ADR-0040) drains a **classify queue** carrying `_ClassifyRequest(stub, body, parser_id)` items — body already fetched and all non-LLM gates already passed on the parser thread. The workers run only the LLM call + verdict handling; they no longer perform body fetch. The shared **Quota Wall** is unchanged. The full prompt (instructions + `{SELF_DESCRIPTION}` + `{MATCH_CRITERIA}` per ADR-0043 amendment + per-listing Header pre-fills + stripped body) is sent as a single combined message via stdin (ADR-0038). Pre-fill bullets render line-level: empty values (`None` or `.strip() == ""`) drop their bullet entirely. The prompt instructs the model to short-circuit on the first failing check — precision-favoring trade. On `ClaudeUsageLimitError` the worker publishes `reset_time` to the **Quota Wall** (ADR-0023, ADR-0040). Malformed response (bad JSON, missing tag, empty `header` or `summary` when `matches: true`) fails that one item — no `seen.json` write, raw output stashed to `<settings-dir>/.runtime-data/failures/malformed/<source>-<url-slug>.md` per ADR-0041, retried next cron tick. Every call writes event + transcript rows to `logs/llm/classify_relevance.events.jsonl` / `.transcripts.jsonl` (ADR-0045 layer-subdir layout; the in-row component identifier stays `llm_classify_relevance`). Runs only on candidates that survive all non-LLM gates (Dedup, **Domain Pre-Filter**, **Freshness Gate**, body fetch, **Content Gate**) and have no prior `seen.json` record. Non-matching candidates get `mark_out_of_domain(stub)` immediately. _Avoid_: filter (ambiguous — see **Domain Pre-Filter**), gate.

**LLM Enricher**: The pipeline stage that owns the classify LLM call site. Per ADR-0047 (amends ADR-0041), body fetch + strip + token-cap + the **Content Gate** all move out — body production returns to **Parser.enrich()**, the **Content Gate** runs on the parser thread. Per ADR-0051, the LLM Enricher receives `(stub, body)` via the **classify queue** (items already gate-checked on the parser thread), runs `LLMExtractor.classify_relevance(item)`, runs the post-LLM arm of the **Freshness Gate** on the LLM-inferred Header date, writes the Card to `extracts.json` on `matches=True`. Single LLM call per candidate produces `{matches, header, summary}` (field renamed per ADR-0043). No httpx client, no body strip, no `body_selector`, no fetch fallback. Redirect-following (formerly an LLM Enricher responsibility per ADR-0041) now lives inside the fallback path in `parsers/body_fetch.py`. _Avoid_: enricher (without "LLM" — overloaded with the parser's `enrich()` method), extractor.

**Freshness Gate**: A gate dropping temporally invalid candidates (ADR-0025, amended by ADR-0041 and ADR-0047). Runs at **three** call sites: post-discover and post-enrich on the parser thread (per ADR-0051; formerly inside the **Gates Bundle**) (`admit_stub(stub) -> bool` — cheap stub-state drops, no LLM cost paid), and a standalone post-LLM arm inside the **LLM Enricher** (`admit(candidate) -> bool` — catches `posted_date` values that only the LLM could infer from body). Drops from both parser-thread arms summed into one `freshness` counter on the **Status Display** per-parser row (ADR-0052). Drop conditions are the same at every site: `posted_date is not None and (anchored_today - posted_date).days > MAX_LISTING_AGE_DAYS`, **or** `deadline is not None and deadline < anchored_today`. `None` on a field is "no signal, don't drop on that field alone"; both `None` → pass (silent no-op, no transcript row). `anchored_today` is the cron-anchored logical date (ADR-0021). Drops write status `expired`; on `matched → expired` the URL's `extracts.json` entry (`{header, summary}` per ADR-0041) is deleted. Per-position transcript gains a `gate_arm: "discover" | "post_enrich" | "post_llm"` field. Log component `pipeline_freshness`. Future-dated `posted_date` (negative age) passes silently. _Avoid_: staleness filter, expiry gate, date filter.

**Content Gate**: A gate called by the parser thread post-enrich (per ADR-0047 — moved from inside the **LLM Enricher** per ADR-0041, originally introduced per ADR-0039; formerly inside the **Gates Bundle**, now called individually per ADR-0051). Drops candidates whose body is empty. Runs only post-enrich (pre-enrich the stub has no body). Owns its full operational protocol — verdict, per-position transcript, per-run aggregate — behind `admit(stripped_body, stub) -> bool` / `emit_run_complete()`. Drops do **not** mark `seen.json`: the URL is re-discovered, re-fetched, and re-checked on the next run, giving the source a chance to publish the body later. Bucket-named (not `EmptyBodyGate`) so future content-validity rules — minimum body length, body-language detection — extend the same module without a rename. Reason enum is open-ended `{passed, empty_body}`. Log component `pipeline_content`; transcript record `url, title, source, passes, reason, body_len`; per-run aggregate `content_considered, content_passed, content_dropped_empty_body`. Effective customer post-ADR-0049 is non-native-enrich parsers (scrape-based) — native enrich paths either return non-empty or skip the stub with a **Failure Report** (ADR-0049), so their output never reaches the Content Gate. _Avoid_: empty-body filter, body validator.

**Domain Pre-Filter**: A gate before the **Relevance Classifier**, dropping **Positions** whose **title** matches any **Negative Keyword** (title-only, blacklist-only, no whitelist — ADR-0019). Substring match, case-insensitive after `normalize()`. Owns its full operational protocol behind `admit(position) -> bool` / `emit_run_complete()`. Drops write status `out_of_domain` (same status the classifier writes for LLM-rejected items, per ADR-0020 renaming). Enumerates all matching keywords internally so it records per-position decisions to `logs/pipeline/prefilter.transcripts.jsonl` and emits a per-keyword end-of-run summary to `logs/pipeline/prefilter.events.jsonl` (ADR-0045 layer-subdir layout; the in-row component identifier stays `pipeline_prefilter`). Transcript record: `url, title, source, passes, reason ∈ {passed, blacklist_drop}, blacklist_matches, title_len`. Aggregate counts *positions matched* (not raw occurrences) and surfaces a `NEGATIVE_KEYWORDS_dead` list of zero-match keywords. Assumes input has been keyword-filtered upstream (parser invariant). _Avoid_: filter (ambiguous — see **Relevance Classifier**), gate, classifier.

**Gates Bundle**: _Retired as a single call site_ per ADR-0051. The four non-LLM gates — **Deduplication** (Run-scoped + Store), **Domain Pre-Filter**, **Freshness Gate**, **Content Gate** — are now invoked individually by the parser thread at their specific pipeline positions. Pre-enrich: Freshness → Dedup → Pre-Filter. Post-enrich: Freshness → Content Gate. Pre-Filter no longer runs post-enrich (redundant — title unchanged by enrich). Freshness drops from both arms summed in the status display. Each gate remains stateless and idempotent; individual gates continue to own their own transcripts and aggregates. _Avoid_: gate runner, gate stack, filter chain.

**EnrichResult**: The dataclass returned by `Parser.enrich(stub)` per ADR-0047. Carries `stub: PositionStub` (possibly with optional fields back-filled by the native enrich path — e.g. Bundesagentur's `/jobdetails` filling in `posted_date` the search index didn't expose), `body: str` (clean text, no HTML), `mode: Literal["native", "fallback"]`. Per ADR-0051, the parser thread threads the returned stub forward (not the input stub) and feeds `body` into the post-enrich gate checks and the classify queue. For native-enrich parsers, `mode` is always `"native"` — the `"fallback"` mode is only produced by parsers whose `enrich()` delegates to `fetch_and_strip` as their primary strategy (ADR-0049). On unrecoverable HTTP failure, `enrich()` raises `EnrichFailedError` — the parser thread catches it, logs the event, increments the `enrich_failed` counter on the parser's status row, and skips the stub without writing to `seen.json` (ADR-0048). _Avoid_: enriched stub (overloaded), enrich payload.

**Quota Wall**: The shared coordination object the **Relevance Classifier**'s parallel worker pool checks before each call (ADR-0040). Implemented as a `threading.Condition` + `reset_time: datetime | None` guarded by the same lock. Surface: `raise_wall(reset_time)` (returns True iff this call was the first to raise the current wall — used to gate the single `quota_sleep` log row), `wait_if_blocked()` (blocks until past the wall; cheap no-op on the common path), `is_active()`. First worker to observe `ClaudeUsageLimitError` publishes `reset_time`; the orchestrator emits exactly one `event=quota_sleep` row per wall raise; remaining workers hit `wait_if_blocked()` at their next iteration and block on the same condition. In-flight calls at wall-raise time are allowed to finish — they will most likely 429 themselves, re-queue, and be sleep-blocked at next dequeue. With `claude_classify_parallelism = 1` the wall degenerates to a single-thread `sleep` (matches ADR-0023's original single-worker semantics). _Avoid_: rate limiter (no rate-shaping; only sleep coordination), barrier (not a one-shot sync point — the wall is repeatedly raisable across a run).

**Match Judge**: The call site that picks the **Daily Top-5** from today's **Pool**. Implemented as a single `LLMExtractor.judge_top_n(candidates)` per run — no wrapper class, no per-item call. Per ADR-0020 (amended by ADR-0041), takes `list[JudgeCandidate]` (stable id + **Header** + **Summary**) and returns up to 5 **Match Verdicts** (`{id, rank}`) — rank-only, no judge-produced summary. The judge ranks on Header + Summary prose; `raw_description` and (retired) `StructuredExtract` do not reach the judge prompt. **Skills** reach the judge via the `{skills}` slot. Per ADR-0043 (supersedes ADR-0016's per-call-site routing; amended post-#535), the judge consumes the same `{SELF_DESCRIPTION}` + `{MATCH_CRITERIA}` profile slots as the classifier (plus its judge-only `{SKILLS}` slot per ADR-0019) — the previous "judge sees match-criteria, classifier sees domain-fit" split retires. On `ClaudeUsageLimitError` orchestrator sleeps and retries (ADR-0023). On any non-quota error the run fails without writing a daily file, and a **Failure Report** is emitted. Runs only on days the **Pool** is non-empty. With ADR-0043 tightening the classifier into a three-check gatekeeper, Pool sizes shrink at source — addressing one of the directions in issue #524.

### Deduplication and run state

**Deduplication**: Skipping **Positions** seen in a previous run or earlier in the current run. Three-tier: an in-run ephemeral URL set (`RunScopedDedup`, obtained from the **Deduplication Store** via `run_scope()` context manager, sits in front of the persistent store to absorb the Cartesian overlap without paying duplicate `enrich()` cost) plus the persistent **Deduplication Store** with two tiers: exact-match on URL, plus exact-match on `(company_lc, title_lc, location_lc)`. The tuple tier fires only when **all three fields are non-`None`** on both sides. When the tuple matches under a new URL (syndicated copy), the store records an alias under the new URL — copying `status` and `first_seen` — so subsequent runs hit the cheap URL tier (ADR-0003). `first_seen` answers "when did this *role* first appear", not "when did this URL first appear". `is_seen` returns a `SeenResult`: `url_hit`/`tuple_hit` skip; `judge_pending` enriches and routes directly into the **Pool** for today's judge call (no classify pass); `miss` processes from scratch. Alias write is performed inside `is_seen`.

**Dedup status enum** (per ADR-0020 / ADR-0022, supersedes the prior set):
- `out_of_domain` — written by **Domain Pre-Filter** (title hit) and **Relevance Classifier** (LLM `matches: false`). Terminal-skip.
- `matched` — written by classifier on `matches: true`, alongside the `{header, summary}` record in `extracts.json` (ADR-0041, supersedes the prior **Structured Extract** shape from ADR-0022). Means "in the **Pool**"; re-discovery routes directly into today's judge candidates.
- `selected_by_judge` — written after judge picks the item and the **Card** is appended+fsynced. Terminal-skip; extract deleted.
- `expired` — written by **Freshness Gate** when `posted_date` exceeds `MAX_LISTING_AGE_DAYS` or `deadline` < anchored date (ADR-0025, amended ADR-0041 — gate runs on both arms). Terminal-skip. Transitions from `matched` (also deletes the `{header, summary}` record from `extracts.json`) or any other status.
- ~~`enrich_failed`~~ — _retired_ per ADR-0048. All per-stub failures are transient — the URL stays unrecorded in `seen.json` and is retried next run. Natural discover falloff bounds the retry tail. Existing `enrich_failed` records in `seen.json` remain inert (short-circuit via URL-tier dedup).
- ~~`external_redirect`~~ — _retired_ per ADR-0041 (which retired ADR-0013 + ADR-0037). Redirects are followed silently inside the **LLM Enricher**.

**Error semantics for `mark_*`** (orchestrator behavior, single-writer Pi):
- **Pre-filter drop** → `mark_out_of_domain(stub)` immediately. No LLM cost.
- **Classifier non-quota error** (per-listing, post ADR-0036/0040): the failing listing's worker raises; that listing is NOT marked in `seen.json`; orchestrator continues with remaining listings in the queue and still calls the judge on the **Pool** as it stands at end of run (ADR-0021). Next cron tick retries the failed URL. Malformed-output failures additionally stash a markdown record to `<settings-dir>/.runtime-data/failures/malformed/` per ADR-0041.
- **Classifier `ClaudeUsageLimitError`** (ADR-0023, ADR-0040): the first worker to observe the 429 publishes the reset time to the **Quota Wall**; all workers sleep until reset+2min (or fallback `next_top_of_hour + 2min`), then resume. The originating listing is re-queued and retried after the wall lifts. No `degraded_reason`; no Failure Report. Cron-anchored day (ADR-0021) handles midnight crossings.
- **Judge non-quota error**: no daily file; **Failure Report** emitted; **Pool** intact (no `selected_by_judge` transitions); tomorrow sees today's pool plus tomorrow's arrivals.
- **Judge `ClaudeUsageLimitError`**: sleep and retry.
- **Per-card append succeeds, then judge processing fails mid-loop**: each card is appended+fsynced+`mark_selected_by_judge` atomically per winner. A failure between cards leaves earlier winners marked, later winners unmarked (back in pool for tomorrow) — bounded.
- **Body fetch raises unrecoverable HTTP error** → stub skipped, URL unrecorded in `seen.json` (transient, retry next run per ADR-0048). For native-enrich parsers (ADR-0049): 401/403, non-retryable 5xx, 3xx redirect, and JSON decode errors mark the parser dead for this run and write a **Failure Report**; 404/400/422 and retries-exhausted skip the stub silently. Empty body from native enrich also writes a Failure Report and skips the stub.
- **HTTP 3xx during body fetch** (ADR-0041, supersedes ADR-0037): the **LLM Enricher**'s HTTP client follows redirects silently up to its configured chain limit. No host classification, no `ExternalRedirect` payload. The destination's HTML is fed into the same strip path. If the destination has no usable body, **Content Gate** drops it (no `seen.json` mark). If the destination is oversized, the raw HTML is stashed to `.runtime-data/failures/oversized/` (no `seen.json` mark).
- **Stripped body over token cap** (ADR-0041): no `seen.json` write; raw HTML stashed to `<settings-dir>/.runtime-data/failures/oversized/<source>-<url-slug>.html`; re-checked next run.
- **LLM output malformed** (empty `header`/`summary` when `matches: true`, missing tag, bad JSON; ADR-0041, field renamed by ADR-0043): no `seen.json` write; a markdown file with the error message, the prompt, and the raw LLM response (or CLI `stderr`/`returncode` when the call failed before a response) is stashed to `<settings-dir>/.runtime-data/failures/malformed/<source>-<url-slug>.md`; re-checked next run.

State lives in `<settings-dir>/.runtime-data/seen.json` (ADR-0046; synced via Syncthing alongside Daily Results Files, ADR-0002), shaped `{url: {company_lc, title_lc, location_lc, status, first_seen}}` (any of the lowercased fields may be `None`); fuzzy index built in-memory at load, not persisted. `DeduplicationStore` exposes four narrow methods (one per live status: `mark_out_of_domain`, `mark_matched` — takes the extract as kwarg, `mark_selected_by_judge`, `mark_expired`). The retired `mark_enrich_failed` (ADR-0048) and `mark_external_redirect` (ADR-0041 supersedes ADR-0013/0037) must not be reintroduced. Pipeline is single-writer at the process level (ADR-0002); the lock inside `DeduplicationStore` (ADR-0014) covers concurrent classify/judge worker writes. Parser threads remain pure producers. **Migration**: ADR-0024 wipes the previous `.seen.json` and the old trio (one-time, prior schema cutover); ADR-0046's rename `.seen.json` → `.runtime-data/seen.json` is a manual `mv` by the operator on upgrade — no auto-move, no guard. _Avoid_: duplicate filtering, URL filtering.

### Display

**Card**: The rendered markdown block per **Daily Top-5** winner, produced by the **Renderer**. Fixed two-block structure per ADR-0042: `# **{rank}:** {Header}` followed by a blank line and `{Summary}` (each terminated by `\n`). `{rank}` is the Judge-assigned 1..5 position; `{Header}` and `{Summary}` are the LLM-authored strings persisted in `extracts.json` (ADR-0041). No placeholder vocabulary beyond rank; no user-tunable layout (ADR-0042 retires `Layout`). _Avoid_: card template, headline.

**Renderer**: A one-line concatenation: `render(rank: int, header: str, summary: str) -> str` returns `f"# **{rank}:** {header}\n\n{summary}\n"`. No `Position` argument, no `MatchVerdict` argument, no `Layout` argument. Per ADR-0042 (supersedes ADR-0004). _Avoid_: formatter, presenter.

**Results File Manager**: The only module that reads or writes the **Daily Results File**. Function-shaped: orchestrator holds the day's `Path` (`<settings-dir>/results/{cron_anchored_date}.md`, ADR-0021) and dispatches. Surface: `ensure_initialized(path)` (`mkdir(parents=True)`; no header — files start empty), `append(path, rendered_block)` (verbatim write + `flush` + `os.fsync`, propagates `OSError`). `data_dir` is the parent of the loaded `config.py` (ADR-0011). _Avoid_: writer, output manager.

**Atomic Write Helper**: A single-function module providing crash-safe atomic overwrite of a file from a `bytes` payload. Interface: `write_atomic(path: Path, payload: bytes) -> None`; raises `OSError` (unwrapped) on any failure. Implementation invariant: opens a `.tmp`-suffix sibling of `path` (named by the helper — callers do not supply a tmp path), performs `os.write` → `os.fsync` → `os.close` → `os.replace` with mode bits `0o644`. A `.tmp` file on disk after a run is always a crash signal — successful writes leave no `.tmp` artefact behind. Encoding is the caller's responsibility; callers pass `payload.encode("utf-8")` (the helper is format-agnostic). No locking: thread-safety stays with the caller; ADR-0002 (single-writer) and ADR-0014 (per-store lock) remain the sole concurrency mechanisms. Used by the **Deduplication Store** and the **Extract Store** from their `_persist` methods. The **Failure Report** writer's weaker `tmp+replace` variant (no `fsync`) is a known future adoption target. _Avoid_: persistence helper (overloaded), file writer (overloaded with the **Results File Manager**'s append idiom — which is not atomic-replace and does not use this helper).

### Observability

**Log Artifacts**: Files under `<settings-dir>/.runtime-data/logs/` (ADR-0046; was `<settings-dir>/logs/`), laid out **by reader** (ADR-0018, layer-subdir layout per ADR-0045). Four types: `<layer>/<comp>.events.jsonl` (one structured row per step; the run-end metrics row that used to live on the retired Run Divider now lands as `event=run_complete` here per ADR-0021); `lifecycle.jsonl` (single shared file at the `logs/` root carrying status-display `registered`/`phase_changed`/`removed` events); `run.log` (single shared file at the `logs/` root carrying tracebacks and `SUMMARY OF SESSION` blocks); `<layer>/<comp>.transcripts.jsonl` (LLM components only — full prompt/response payloads). Per-component files live under `logs/parser/`, `logs/llm/`, `logs/pipeline/`; the bare component name is the filename, the subdir conveys the layer. Component identifiers themselves stay layer-prefixed (`parser_`, `llm_`, `pipeline_`) in every event/lifecycle row, in `run.log` headers, and in Status Display labels — the subdir replaces the prefix only on disk. For LLM code, **component is the LLM-stage row**, not the implementation class — `llm_classify_relevance` for the per-item classifier and `llm_judge_match` for the end-of-run top-N judge (the identifier predates ADR-0020's switch from per-item `judge_match` to single-call `judge_top_n` and is kept for log-file continuity). _Avoid_: log file (without qualifier), debug log.

**Run Log**: The per-run instance that writes the **Log Artifacts** to `<settings-dir>/.runtime-data/logs/` (ADR-0046). Constructed once at orchestrator entry from `cfg.logs_dir`; threaded as a kwarg into every component that emits log events (`RunMetrics`, `ClaudeExtractor`, the status-display core, the outbound dispatcher, every **Parser** via context-manager construction which forwards into its `ParserHttp`). Owns the directory path, file-naming convention, JSONL envelope shape, and `=== <component>  <ts>  <kind> ===` header convention in `run.log`. Filesystem errors propagate as `OSError`. Safe to share across threads — each method opens, writes, closes per call. _Avoid_: log writer, logger (overloaded with `logging.Logger`).

**Status Display**: A live, in-process view of pipeline progress. Ephemeral — exists only while `run()` executes. A `StatusDisplay` Protocol with two implementations: `RichStatusDisplay` (`rich.Live` table repainting at ~4 Hz, when `sys.stdout.isatty()`) and `PlainStatusDisplay` (line-by-line `print`, Pi cron). Per ADR-0052, every stage reports uniform counters: **queued, dropped, forwarded**. Per-parser rows show `K discovered · [per-gate named drop counters if nonzero] · [enrich_failed if nonzero] · N forwarded`; gate drop counters name the gate that dropped (e.g. `3 freshness · 30 dedup`). Zero-drop counters hidden. Freshness drops from pre-enrich and post-enrich arms summed into one counter. Parsers without `has_native_enrich` skip the `enrich_failed` counter. `llm_classify_relevance` row uses the same queued/dropped/forwarded model. Judge has no persistent status row — outcome logged as a terminal message (ADR-0052). Overall `pipeline` phase row and transient `startup` row remain. Per-gate rows (`pipeline_dedup`, `pipeline_prefilter`, `pipeline_freshness`, `pipeline_content`) and `llm_judge_match` row retired. Body strings formatted by **RunMetrics**. `register()`/`update_phase()` also emit to `lifecycle.jsonl` (ADR-0018). _Avoid_: progress bar (implies known total), TUI, dashboard.

### Sources & extraction

**Source**: A configured job board or API in `Config.SOURCES`, declared as a `SourceEntry` carrying a **Parser Type**. A **Source** is a *config entry* (what to search); a **Parser** is the *module* (how). One-to-one in v1. _Avoid_: site, board.

**ParserHttp**: The per-parser HTTP layer at `src/application_pipeline/parsers/http.py`. One instance per **Parser** (constructed and owned inside the parser's context-manager scope), wrapping a single `httpx.Client` with per-host pacing (`REQUEST_PACING = 0.5s` via an internal `_Throttle`), retry-with-backoff on a closed set of retryable statuses (`{429, 502, 503, 504}`, capped by `MAX_RETRIES = 3` with exponential backoff `1s → 2s → … → 8s`), and a typed error taxonomy (`HttpRetryError`, `HttpStubNotRetryableError`, `HttpParserFatalError`, `HttpNotRetryableError`, `HttpRedirectResponse` — per-stub vs parser-fatal split per issue #379/#380). `RunLog` is injected at construction and forwarded on every request. Parsers go through `ParserHttp` for discover-phase fetches today; under ADR-0047 the shared `parsers/body_fetch.py` fallback helper also takes `ParserHttp` as a kwarg, so the same retry/pacing/taxonomy applies to body fetch via the fallback path. Native `enrich()` paths (e.g. Bundesagentur's `/jobdetails` API) also dispatch through their parser's `ParserHttp`. _Avoid_: http client (overloaded with raw `httpx.Client`), fetcher (overloaded with **Parser**).

**Parser**: A Python module in `src/application_pipeline/parsers/`. Two-method API per ADR-0047 (amends ADR-0041's single-phase shape): `discover(query) -> Iterable[PositionStub]` and `enrich(stub) -> EnrichResult`. Each parser is a context manager owning a single `httpx.Client`; no shared mutable module-level state (ADR-0005, amended by ADR-0051 — parser threads now perform `enrich()` I/O inline with discovery, interleaved on the same thread). The orchestrator owns Cartesian expansion of `KEYWORDS × LOCATIONS [+ remote]` and emits one `discover(ParserQuery(keyword, location))` per combination; `location` is sealed **Location** (`City(name) | Remote`). Each parser declares its **Location Coverage** as module-level symbols (ADR-0012). Per ADR-0051, each parser thread runs the full pipeline: `discover → per-gate checks (Freshness → Dedup → Pre-Filter) → enrich → per-gate checks (Freshness → Content Gate) → enqueue to classify queue`. Only stubs that pass all gates and have a body reach the classify queue. Parsers own both discover-phase per-host pacing AND body-fetch pacing — the latter via either a source-specific native call or the shared `parsers/body_fetch.py` helper (the parser's `enrich()` chooses). Parsers declaring `has_native_enrich = True` use their native path exclusively — no fallback to `fetch_and_strip` (ADR-0049); if the native path fails or returns an empty body, the stub is skipped with a **Failure Report**. Parsers with `has_native_enrich = False` use `fetch_and_strip` as their primary (and only) enrich strategy. `body_selector: str | None` is parser-private state consumed only by `fetch_and_strip` — no longer a Protocol attribute. `has_native_enrich: bool` (module-level constant, default `False`) gates **Status Display** counter visibility. In-parser dedup (e.g. reference-number sets) is retired per ADR-0050 — the **Deduplication Store** is the sole dedup authority. **Keyword-match invariant** (ADR-0019): every stub yielded from `discover()` is expected to match its `ParserQuery.keyword` in the title. _Avoid_: scraper, fetcher.

**Position Stub**: The result of `Parser.discover()` — `url, title, source, posted_date?, company?, location?`. Required: `url, title, source`. Optional (`| None`): `posted_date: date`, `company: str`, `location: str` — drives the post-discover per-gate checks (Freshness, Dedup, Pre-Filter) and feeds Header pre-fills into the **LLM Enricher** call. A parser's `enrich()` may return an updated stub with any of these optional fields back-filled from the native enrich path (e.g. Bundesagentur's `/jobdetails` exposes `posted_date` the search index didn't); the parser thread threads the updated stub forward into the post-enrich gate checks (Freshness, Content Gate) and the classify queue. _Avoid_: preview, summary.

**External Redirect**: _Retired_ per ADR-0041 (which supersedes ADR-0013 and ADR-0037). Redirects are followed silently during body fetch — under ADR-0047 the redirect-follow lives in the shared `parsers/body_fetch.py` fallback helper (was inside the **LLM Enricher** under ADR-0041). _Avoid_: do not reintroduce as a payload type; the `external_redirect` `SeenStatus` and `external_redirects` counter are also gone.

**Parser Type**: The string on a `SourceEntry` identifying which **Parser** to use — maps directly to a filename in `parsers/`. _Avoid_: adapter.

**Location**: Geographic slot the orchestrator passes via `ParserQuery.location`. Sealed sum type: `City(name: str)` for a named place from `Config.LOCATIONS`, `Remote` for the remote-only slot when `INCLUDE_REMOTE=True`. _Avoid_: place, geo.

**City**: The `City(name)` arm. `name` is the user-typed string, normalized via `normalize()` before lookup. No central catalog — resolvable city names are the dynamic union of configured **Sources**' `serves()` predicates. _Avoid_: town, place.

**Remote**: The `Remote` arm. Each **Parser** decides what `Remote` means for its **Source** via `remote_wire()`. _Avoid_: homeoffice, work-from-home (when the type variant is meant).

**Location Coverage**: The module-level **Protocol** every **Parser** satisfies — `serves(name: str) -> bool`, `to_wire(name: str) -> str`, `serves_remote: bool`, `remote_wire() -> Any`. Each parser is local authority. **Config Loader** validates `LOCATIONS` at load time and rejects unresolvable entries with `ConfigError`. Per ADR-0012. _Avoid_: location map, slug table.

**LLM Extractor**: A Protocol with two methods — `classify_relevance(item: ClassifyItem) -> (RelevanceVerdict, CallUsage)` and `judge_top_n(candidates: list[JudgeCandidate]) -> (list[MatchVerdict], CallUsage)`. Per ADR-0041: `ClassifyItem` carries `title`, `raw_description` (stripped body), and any parser-pre-filled `PositionStub` Header fields. `RelevanceVerdict` is `{matches: bool, header: str | None, summary: str | None}` (header/summary non-None iff `matches: true`; ADR-0043). `JudgeCandidate` carries stable id, **Header**, **Summary**, and the `PositionStub` fields for tie-breaking. `MatchVerdict` is `{id, rank}` — rank-only (ADR-0041); the prior `matched`/`missing`/`summary` fields retire. Hardcoded prompts live in `src/application_pipeline/templates/prompts/`; **Triage Profile** injected via `{SELF_DESCRIPTION}` + `{MATCH_CRITERIA}` slots at load time; **Skills** baked into judge's `{SKILLS}` slot at load time (ADR-0043 amendment). The v1 production implementation is `ClaudeExtractor` shelling out to **Claude Code CLI** (`claude -p - --output-format json --model <alias>` + `--effort <level>` where applicable, plus `--no-session-persistence --disable-slash-commands --tools "" --setting-sources user` per ADR-0038) as a fresh subprocess per call. The full combined prompt is sent as a single message via stdin (ADR-0038); `--system-prompt` is not used. Per ADR-0015: `haiku` for the classifier, `haiku` with `--effort medium` for the judge (model choice may be revisited under the more-capable-LLM regime). Structured outputs wrapped in `<verdict>` / `<verdicts>` tags and parsed via **Agent Output Protocol** (ADR-0015). On `ClaudeUsageLimitError` the implementation parses the 429 reset time and raises `ClaudeUsageLimitError(reset_time)` so the orchestrator can sleep (ADR-0023). _Avoid_: LLM, model (without qualifier).

**Agent Output Protocol**: A project-agnostic pure module under `application_pipeline/llm/` that extracts a structured JSON payload from a CLI response whose `result` carries free-form text with payload wrapped in a semantic XML tag (ADR-0015). Exposes `extract_json_block(text: str, tag: str) -> Any` + `AgentOutputProtocolError(kind: "tag_missing" | "json_malformed")`. Finds rightmost `</tag>`, walks back through preceding `<tag>` openings, strips an optional surrounding markdown fence, then `json.loads`. Returns the first candidate that parses; raises on no tag / no parse. The **LLM Extractor** catches at its call-site boundary and re-raises as `ExtractorBatchMalformedError`/`ExtractorMalformedJSONError`. _Avoid_: output parser (too generic), response handler (overloaded with HTTP).

**Pagination**: Successive page fetches per **Source**, preferably ordered newest-first by `posted_date`, until the source signals no next page (empty results). Per ADR-0017 there is no dedup-driven discover early-stop. `SKIP_AND_END_QUERY` (ADR-0007) is only emitted on `run_state.is_aborted`. ~~`max_results`~~ cap is _retired_ per ADR-0050 — parsers paginate to source exhaustion.

### Invocation

The package is distributed via PyPI (ADR-0027). Hosts install into a project-local `.venv` (`python3 -m venv .venv && .venv/bin/pip install application-pipeline`), bootstrap the settings folder with `application-pipeline init`, and arm cron with `bash application-pipeline/setup/cron-install.sh`. **Home dir is hardcoded to `<cwd>/application-pipeline/`** — no env-var override, no auto-discovery (ADR-0029, mirrors pycastle's CWD-relative pattern). The three subcommands take no settings-dir argument:

- `application-pipeline run` reads `<cwd>/application-pipeline/config.py`.
- `application-pipeline init [--refresh]` seeds into `<cwd>/application-pipeline/`.
- `application-pipeline compile-cv <app_dir>` compiles a per-listing draft. Reads `<app_dir>/cv.tex` (slot-map written by `/write-cv` — see **CV Slot-Map** and **CV Skeleton**), substitutes its slots into the package's `cv_template.tex`, `\input`s `cv/facts.tex` from `<cwd>/application-pipeline/user-info/`, writes the result to `<app_dir>/.build/` and runs `pdflatex` for the three `\BUILD` variants. Windows paths are normalized to POSIX (`as_posix()`) before injection into `\def\CvDataDir{...}` (renamed from `\UserDataDir` per ADR-0032; now points at `user-info/cv/`) to avoid TeX swallowing backslashes as control sequences (ADR-0030).

`run` and `compile-cv` fail loud-and-fast (exit 2, named error) at the CLI boundary if `<cwd>/application-pipeline/config.py` is missing — the message names both fixes ("did you forget to cd, or run init?"). Cron fires **weekdays at 00:30 local** (ADR-0024, amended). Each tick runs the seeded `setup/cron.sh`, which `cd`s to the project root via `$(dirname "$0")/../..` (self-locating; cron line carries only the absolute path to `cron.sh`) before invoking: `pip install --upgrade application-pipeline` (×2 for CDN propagation), `application-pipeline init --refresh`, then `application-pipeline run`. **`init` semantics** (refined post-ADR-0011, amended ADR-0042, expanded ADR-0044): plain `init` seeds-if-missing for everything; `init --refresh` overwrites package-owned subtrees unconditionally (`setup/*.sh` under `application-pipeline/`, the `cv-template/` directory under `application-pipeline/`, and the package-owned skill directories `_shared/`, `analyse-listing/`, `iterate-cv/`, `write-cv/` under `.claude/skills/`), **deletes** `layout.py` if present (ADR-0042) and the obsolete `application-pipeline/skills/` dir if present (ADR-0044 relocates `cv_skeleton.tex` to `cv-template/`), and seeds-if-missing for everything else (`config.py`, `user-info/*`, `.gitignore` containing `.runtime-data/` per ADR-0046 — never overwritten). **Refresh never touches `<cwd>/application-pipeline/.runtime-data/`** (same boundary as `user-info/`: pipeline-written state is never deleted by an upgrade). Refresh never deletes unknown files inside package-owned skill dirs (operator notes survive) and never touches non-package skill dirs (user-added skills like `<cwd>/.claude/skills/my-skill/` survive). The templates tree is organised into routing buckets (ADR-0044): `templates/application-pipeline/...` seeds to `<cwd>/application-pipeline/`, `templates/claude/skills/...` seeds to `<cwd>/.claude/skills/`. User-authored files are safe across upgrades; new global template files added in a release self-heal onto the host. A global `flock` at `<cwd>/application-pipeline/.runtime-data/.cron.lock` (ADR-0046; modelled on pycastle's pattern) serialises overlapping ticks; a run that sleeps through a quota window (ADR-0023) may overlap the next cron fire, which the flock silently waits out — absence of the next weekday's daily file is the operator signal. The laptop is used only during development to smoke-test individual **Parsers** in isolation. Note: `pycastle/` in this repo is an unrelated RALPH Loop coding-agent plugin used to *build* this project; its quota-handling logic (`pycastle/services/claude_service.py:88-145`) is the reference port for ADR-0023's parse-and-sleep behavior, and its `setup/*.sh` scripts are the reference shape for ADR-0027's seeded cron wrappers.

## Relationships

- A listing is written into a **Daily Results File** only if it passes (in order, per ADR-0047/ADR-0051): on the parser thread — **Freshness Gate** (stub `posted_date`), **Dedup**, **Domain Pre-Filter** (title), **Parser.enrich()** (body fetch via native source path or shared fallback helper, may back-fill stub fields), **Freshness Gate** (post-enrich re-run), **Content Gate** (body) — then enqueued to the classify queue, where classify workers run the **Relevance Classifier** LLM call (status → `matched`, `{header, summary}` persisted to `extracts.json`), the standalone post-LLM **Freshness Gate** arm (catches LLM-inferred `posted_date`), then picked into the **Daily Top-5** by the **Match Judge** (status → `selected_by_judge`). The post-LLM **Freshness Gate** arm and the **Content Gate** both re-run on Pool re-discovery, so pool items can transition `matched → expired` (deletes the extract record) or be re-checked when the body is missing.
- The **Pool** is the set of `status == matched` URLs re-discovered by a parser in the current run — an emergent property of `seen.json` + today's parse output, not a stored data structure.
- The **Match Judge** runs once per run on the **Pool**, takes **Header** + **Summary** strings per candidate plus construction-bound **Skills** and the injected **Triage Profile**, returns up to 5 **Match Verdicts** carrying only `{id, rank}` (ADR-0041).
- The **Triage Profile** reaches the LLM via two named slots in the hardcoded prompts (`{SELF_DESCRIPTION}`, `{MATCH_CRITERIA}` — per ADR-0043 amendment, replacing ADR-0016's single `{USER_INFO}` slot); **Skills** are baked into the judge prompt's `{SKILLS}` slot at `load_prompts` time (judge only); `NEGATIVE_KEYWORDS` reaches the **Domain Pre-Filter** directly (ADR-0019).
