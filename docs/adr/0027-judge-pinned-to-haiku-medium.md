# Match Judge pinned to `haiku --effort medium`

The **Match Judge** call site's `--model` pin moves from `sonnet` to `haiku`, keeping `--effort medium`. The **Relevance Classifier** stays on plain `haiku` (no `--effort`). The asymmetric pin shape from ADR-0018 is preserved: model and effort live as `ClaudeExtractor` module-level constants, not as fields on the user-editable `Config`. This ADR updates the model-pin portion of ADR-0018; ADR-0018's tag-anchored output protocol, fence-stripping recovery, prompt structure, and forensics taxonomy all stand unchanged.

## Why

- **Cost is dominated by the judge call.** Over a representative window (`synched/logs/claude_extractor.events.jsonl`, 11 classify batches + 110 judge calls), the judge accounted for $5.19 (60%) of the $8.68 total Claude spend, at ~$0.047 per call. Classify was $3.49 (40%) at ~$0.003 per item. Per item, judge cost ~15× classify cost. Moving judge to Haiku is the single largest cost lever available; nothing else on the LLM side comes close.
- **The user explicitly accepted the quality risk in the token-cost grilling (issue #319).** The judge produces structured output — `tier`, open-vocabulary `matched`/`missing`, 2–3 sentence `summary` — that does exercise actual reasoning. Stepping down the model class is the riskier of the two cuts available (model class vs reasoning effort). The user accepted this trade as part of #319, with the observation that the dominant cost saving is the model-class step and effort can be revisited separately if quality holds.
- **Keep `--effort medium` for now to avoid confounding two variables.** Dropping both the model class and the effort flag in the same change makes a quality regression undiagnosable. With `--effort medium` preserved, a verdict-quality drop after this lands is attributable to the model-class step alone. Dropping `--effort` is a clean follow-up slice once a few runs of verdicts have been observed.
- **Classify stays asymmetric (plain `haiku`, no effort).** ADR-0018's per-call-site rationale stands: classify is a per-item boolean decision over 100 items per batch — reasoning depth is wasted budget on a binary task. The cost picture explicitly shows no classify quality complaint, so the symmetric "haiku-medium everywhere" alternative was rejected. Adding `--effort` to a working, cheap call site to satisfy symmetry contradicts the goal of this change.
- **The asymmetric pin shape is by design.** ADR-0018 chose per-call-site constants over Config fields specifically so call-site reasoning load can drive the choice — classify and judge have different reasoning loads, so they get different pins. This ADR exercises that flexibility rather than fighting it.
- **The prompt-cache contract is preserved.** The static prompt prefix at the **Match Judge** call site is unchanged. The first run after deploy pays a one-time cache miss because the model class changed; subsequent runs cache normally. The cache-read tokens recorded on the **Run Divider** will show the expected single-run dip then recovery — operator-visible proof.

## Considered alternatives

- **Move judge to plain `haiku` (no `--effort`).** Rejected on confounding grounds (see above). Plain haiku is the natural follow-up slice if a few runs of `haiku --effort medium` verdicts hold up quality-wise. Landing both cuts in one change would make a green-rate drop undiagnosable.
- **Align both call sites to `haiku --effort medium`.** Rejected. Adds reasoning effort to a classify site that has no quality complaint, raising its cost for no benefit. The Config table at issue #319 shows classify at $0.003/item with no operator-visible quality problem; lifting that price to satisfy "one rule" symmetry contradicts the cost-cut goal. The pinned-per-call-site shape from ADR-0018 already encodes "asymmetric is fine".
- **Surface model + effort to `Config` so the operator can A/B.** Rejected for the same reasons as ADR-0018: conflates search-shape preferences (Syncthing-edited, user-driven cadence) with call-site engineering decisions (code change, PR review). The risk of silent quality/cost regressions is real — even an experienced operator can flip a Config field without realising it changes verdict shape.
- **Move judge to a yet-cheaper open-weight model run on the Pi locally.** Out of scope for this ADR. ADR-0001 (local Ollama as LLM backend) was retired in favour of the Claude CLI for v1; revisiting that decision is its own architectural conversation.
- **Skip the model-pin change and instead truncate `raw_description` aggressively.** Considered as parallel work (boilerplate-stripping is a separate slice, blocked on #318). Aggressive truncation at Sonnet rates is dominated by the model-class step at any plausible truncation ratio — Sonnet-on-half-the-text still costs more than Haiku-on-full-text per call. Truncation remains a follow-up cleanup, not a substitute.
- **Add a quality-regression detector before landing.** Considered and deferred. The user's stated acceptance of the quality risk in #319 plus the existence of the **Results File** tier distribution as a natural eyeball signal makes a formal detector premature. If a future slice raises the cost-cut ambition further (e.g. dropping effort), revisit then.

## Consequences

- **`ClaudeExtractor` constants change.** `_JUDGE_MODEL = "sonnet"` becomes `_JUDGE_MODEL = "haiku"`. `_JUDGE_EFFORT = "medium"` is unchanged. `_CLASSIFY_MODEL = "haiku"` is unchanged; classify continues to omit `--effort`.
- **Expected cost impact.** Judge per-call cost drops by roughly an order of magnitude (sonnet→haiku price ratio). Estimated total run cost falls from ~$8.68 to ~$4.01 even before the prefilter cleanup (ADR-0026) or boilerplate-stripping (issue #319 slice A) lands. The exact ratio depends on Anthropic's pricing at the time of deploy.
- **Verdict quality is the operator-monitored signal.** The Run Divider continues to record per-run cost and token totals; the **Results File** tier distribution (`green.md` / `amber.md` / `red.md` line counts and Run Divider cadence) is the natural eyeball signal for regression. A noticeable drop in green-tier verdicts in the first few runs is the canary; reverting is a one-line code change.
- **Tests and fixtures.** Any test fixture that asserts `--model sonnet` on the judge call site is updated to `--model haiku`. The `--effort medium` flag continues to appear on judge calls; classify continues to call without `--effort`.
- **CONTEXT.md updates.** The **LLM Extractor** entry's call-shape line moves from `sonnet with --effort medium for the Match Judge` to `haiku with --effort medium for the Match Judge`. No other domain term changes.
- **ADR-0018 is updated, not formally superseded.** Its tag-wrapped output convention, fence-strip recovery, agent output protocol module, prompt example structure, prompt-cache contract, transcripts forensics taxonomy, and invoker/extractor boundary all remain authoritative. Only the `_JUDGE_MODEL` constant value moves; readers should treat ADR-0027 as the authoritative source for the judge model pin going forward (mirroring how ADR-0018 itself updated ADR-0016's argv-shape line without formally superseding it).
- **No operator-visible CLI or config change.** The cron command is unchanged; the user's `synched/config.py` is unchanged; the **Run Divider** field set is unchanged. The only externally visible difference is the cost step-down recorded on the **Run Divider** starting on the first run after deploy.
- **Follow-up slices.** Dropping `--effort` from the judge pin (becoming plain `haiku`) is parked as a separate slice if verdict quality holds over a few runs. Aligning classify to use `--effort medium` is explicitly rejected by this ADR and is not a parked follow-up.
