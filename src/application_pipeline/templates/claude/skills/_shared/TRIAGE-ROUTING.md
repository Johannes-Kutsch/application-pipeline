# Triage-Profil-Routing

<triage-routing>
| Datei | Verwenden für |
|---|---|
| `match-criteria.md` | breite Domain-In/Out-Signale **und** konkrete Fit-Schwellen — „diese Art Rolle sollte gar nicht erst shortlisted werden" *oder* „Domain stimmt, aber eine harte Anforderung kippt"; bevorzugt für Miss bei harten Anforderungen |
| `self-description.md` | „es geht um wer ich bin, nicht was ich will" — Identität/Werte |
| `writing-style.md` | Stimm-/Phrasing-**Regeln** **und** Cover-Strategie pro Slot (Fact-Cross-Check, Pivot-Treiber-Wahl, Hook-Verteilung, Meta-Beleg-Disziplin) — One-Liner bzw. knappe Bullets, deklarativ, keine Exemplare. "Vermeide X" → Don't-Zeile, "mach immer Y" → Do-Zeile, "Inhalts-/Bogen-Regel pro Slot" → Bullet in `## Cover-Strategie`. |
| `positive-exemplars.md` | Stil-**Vorbilder** als Bullet-Snippets aus handgeschriebenen Briefen. "Der Ton von Brief X war besser" → neue Tonangabe-Annotation oder neues Vorbild. **Keine Negativ-Exemplare**, weder hier noch in `writing-style.md`. |
</triage-routing>

<hard-rules>
- Schreibzugriffe auf Profil-Files passieren inline im selben Turn, in dem das Signal aufkommt. „Still" heißt: kein Bestätigungstext im Antwort-Text, aber der Edit-Call wurde gemacht.
- **Conservative Promotion:** nur einen Bullet schreiben, wenn das Signal im aktuellen Turn eindeutig ist. Mehrdeutige Kandidaten als internen Merker im Modell-Kontext behalten (nicht im Antwort-Text sichtbar) und im nächsten Turn neu prüfen, ob sie inzwischen eindeutig wurden.
- Bei semantischem Widerspruch oder Verfeinerung: existierenden Bullet ersetzen, nicht anhängen.
- Bullet-Stil: extrem knapp, Grammatik zugunsten Knappheit opfern (ADR-0026), Deutsch, keine Prosa-Absätze.
</hard-rules>
